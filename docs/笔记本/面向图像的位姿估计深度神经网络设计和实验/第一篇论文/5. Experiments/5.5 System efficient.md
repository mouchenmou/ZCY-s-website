在前面，作者炫耀了精度、鲁棒性、迁移学习。但在工程落地层面，老板和产品经理最关心的永远是：这东西跑得快吗？占内存吗？手机上能用吗？

这最后一节，作者用大写加粗的图表告诉你：传统算法是“重资产”，而我的 PoseNet 是极致的“轻量级”！

我们依然分三刀，把这张图表里极其夸张的性能碾压剥开：

### 第一刀：降维打击的 $O(1)$ 与 $O(n^2)$

> **原文：** These values are independent of the number of training samples in the system while metric localization scales $O(n^2)$ with training data size.
> 
> **大白话：** PoseNet 的时间和内存开销，与系统里训练数据的多少毫无关系；而传统的 SIFT 定位算法，会随着数据量的增加呈 $O(n^2)$爆炸式增长。

**硬核底层逻辑：**

- **传统 SIFT：** 它就像一个死记硬背的图书管理员。剑桥大学有多大，他就得建多大的 3D 点云数据库。当你想算坐标时，他必须拿你拍的照片，跟数据库里的几百万个特征点一一对比。所以，地图越大（X轴往右走），他算得越慢（时间飙升到 120秒），脑子越不够用（内存飙升到 10GB）！这就是致命的 $O(n^2)$。
    
- **PoseNet：** 它就像一个悟出地图真理的老司机。它把剑桥大学所有的空间规律，全部压缩、内化到了神经网络的权重（Weights）里。一旦训练完成，不管你是记住了 10 张图还是 10 万张图，神经网络的层数和参数量是完全固定死的。
    
    所以，每次出题，它只需要执行一次固定的矩阵乘法。这就是极其优雅的 **$O(1)$。
    

### 第二刀：图表上的“贴地飞行”（视觉嘲讽）

> **原文：** Our network is very scalable, as it only takes 50 MB to store the weights, and 5ms to compute each pose...
> 
> **大白话：** 我们的网络扩展性极强，仅仅需要 50 MB 来存储模型权重，仅仅需要 5 毫秒就能算出一个坐标。

**硬核看图 13：**

你仔细看那两条红线，它们在哪？它们完完全全、死死地贴在了 X 轴上！

- 在时间图里，蓝线窜到了 120 秒，而红线的 5 毫秒在图上根本画不出来，看起来就像是 0。
    
- 在内存图里，蓝线窜到了 10 GB，而红线的 50 MB 看起来也像是 0。
    

这在学术界叫做“视觉嘲讽。作者故意把坐标轴拉得巨大，就是为了告诉你：“在我的 PoseNet 面前，传统算法慢得像蜗牛，笨重得像大象。而我，轻量到可以塞进任何一部破手机里实时运行。

### 第三刀：为什么“抄答案”也不行？（解释绿线）

> **原文：** For comparison matching to the convnet nearest neighbour is also shown. This requires storing feature vectors for each training frame, then perform a linear search to find the nearest neighbour...
> 
> **大白话：** 作为对比，我们也展示了用 CNN 找最近邻（抄答案）的表现。这需要你把所有训练图的特征向量都存下来，然后用线性搜索去挨个比对。

**硬核剖析：**

你还记得图里那条绿线吗？这就是我们之前反复提到的“笨办法”（用网络提取特征，然后去题库里找最像的图）。

虽然它比 SIFT好一点，但它的时间和内存依然是一条斜向上的直线（线性增长）。因为题库越大，你需要存的特征就越多，比对的时间就越长。

这也进一步反证了：PoseNet 的红线之所以能绝对平坦，是因为它真正把题库给“融化”吸收了，而不是把题库带进了考场。
