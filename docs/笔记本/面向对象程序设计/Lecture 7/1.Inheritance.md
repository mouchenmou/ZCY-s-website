    - `class DVD : public Item {...}:`
        - 表示`DVD`类也公开继承自`Item`类

    - 图中的注释指出了引入继承后的代码修改策略

## Advantages of inheritance

1. Avoiding code duplication（避免代码重复）
    - 将通用属性和方法定义在父类`Item`中，子类就不用重复编写这些代码（如`title`, `comment`）
2. Code reuse（代码重用）
    - 子类直接继承并使用父类中已有的功能。例如，`CD`和`DVD`都可以直接使用`Item`中的`setComment()`方法。
3. Easier maintenance（更易于维护）
    - 如果需要修改通用功能，只需在父类中修改一次，这样所有的子类都会更新，避免了在多个文件中修改相同代码的麻烦。
4. Extendibility（可扩展性）
    - 可以在不修改现有核心代码的情况下，轻松添加新的功能或数据类型。

## Example

!!! example "下面给出一段示例"
    ### Declare an Employee class
    ```c++
        class Employee{
        public:
            Employee(const std::string& name, const std::string& ssn);
            const std::string& get_name() const;
            void print(std::ostream& out) const;
            void print(std::ostream& out, const std::string& msg) const;
        protected:
            std::string m_name;
            std::string m_ssn;
        };
    ```
    !!! explanation "Question 1: 为什么函数输入使用`&`呢"
        1. 效率：传入`std::string`对象时，如果使用传值(不带`&`)，c++会创建一个完整的副本。而使用`&`意味着函数直接操作原始对象，避免了创建副本所带来的时间和内存开销，这对于像`std::string`这样可能包含大量字符串的大对象尤其重要。
        2. 安全性：
            - `Employee`的输入参数`const std::string& name`：`const`与引用`&`结合，告诉编译器**“我只信啊引用这个对象，请不要允许我在函数内部修改它”**。这保证了函数是**只读**的.
            - 返回值`const std::string& get_name() const`：返回一个`const`的引用到`m_name`成员变量，既保证了效率，又保证了封装性和安全性（因为`const`引用，外部不能通过这个引用来修改`Employee`对象的私有成员`m_name`）
    
    !!! explanation "Question 2: `const std::string& get_name() const;`这句话为什么要用两个`const`"
         首先，get_name()这个函数的前面有一个`&`修饰符，因此这个函数返回的值是一个引用。
         
         1. 第一个`const`:
             - 作用对象：函数返回的引用
             - 含义：它保证了函数返回给调用者的`std::string&`是不可修改的（只读）
             - 效果：这意味着当你调用这个函数的时候，你不能通过返回值来修改原始的`Employee`对象内部存储的姓名。
                 - 错误示例:`employee.get_name() = "New name"`
         2. 第二个`const`:限定函数本身
             - 作用对象：函数所属的`Employee`对象（即this指针所指向的对象）
             - 含义：它保证了这个成员函数是一个**常量成员函数**(`const`类型的成员函数)。
             - 效果：函数内部不能修改调用该函数的对象的任何成员变量（如`m_name`或`m_ssn`）

    ### Employee member functions
    ```c++
        inline const std::string& Employee::get_name() const{
            return m_name;
        }
        inline void Employee::print(std::ostream& out) const{
            out << m_name << endl;
            out << m_ssn << endl;
        }
        inline void Employee::print(std::ostream& out, const std::string& msg) const{
            out << msg << endl;
            print(out);
        }
    ```
    !!! explanation "Question 1: ostream是什么的东西"
        `ostream`是c++标准库中`I/O`系统的核心组件，用于输出操作
        
        - 含义：将一种数据从程序传输到外部设备的抽象概念
        - 功能：它提供了所有流对象都有的输出操作
            - `std::cout`: 连接到标准输出设备（通常是终端/控制台）
            - `std::cerr`: 连接到标准错误流
            - `std::clog`: 连接到标准日志流
            - `std::ofstream`: 连接到文件
    
    !!! explanation "Question 2: 这里为什么是用的是`out`而非`cout`"
        `out`只是我们定义的一个数据类型，它是`ostream`类型的引用，跟`int &a = b;`定义了一个`int`类型的引用是一个道理。
        ```c++
            如果我声明函数时把函数写成这样：
            inline void Employee::print(std::ostream& huangjiexi) const;
            
            那么定义函数时就应该写成：
            inline void Employee::print(std::ostream& huangjiexi) const{
                huangjiexi << m_name << endl;
                huangjiexi << m_ssn << endl;
        }
        ```
        
    !!! explanation "Question 3: 用`ostream`定义的输出最后会输出到哪里呢？"
        #### 之前存在的疑惑：像`cout`就是能够把我们要输出的内容打印到屏幕上，那么`ostream`又能把数据打印到哪里呢？
        
        #### 解答：`msg`被输出到了程序员在调用`print`函数时指定的那个输出目标
        
        1. `out`本身不是一个固定的输出目标（比如屏幕）
        2. `out`是一个接口，它在等待被连接到实际的输出通道
        
        !!! tips "用函数`print(std::ostream& out, std::string& msg)`来举几个例子"
        上面的解释可能比较晦涩难懂，下面的几个例子比较好理解
        假设一个`Employee`对象叫`alice`
            ```c++
                //情况1：输出到屏幕
                alice.print(std::cout, "黄杰曦是弱智");
                
                //情况2：输出到错误日志
                alice.print(std::cerr, "黄杰曦是脑抽");
                
                //情况3：输出到文件
                std::ofstream file("data.txt");
                alice.print(file, "黄杰曦我男神")
            ```
