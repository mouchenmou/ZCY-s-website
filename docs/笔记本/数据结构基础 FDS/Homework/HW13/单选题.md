### 2-1

During the sorting, processing every element which is not yet at its final position is called a "run". Which of the following cannot be the result after the second run of quicksort?

A. 5, 2, 16, 12, 28, 60, 32, 72

B. 2, 16, 5, 28, 12, 60, 32, 72

C. 2, 12, 16, 5, 28, 32, 72, 60

D. 5, 2, 12, 28, 16, 32, 72, 60

??? success "答案与解析"
    **答案：D**

    **核心原理解析**

    在快速排序（Quicksort）中，每一"趟"（run）的核心操作是选取一个基准值（pivot），并将数组划分（partition）。一趟操作完成后，会产生一个非常重要的必然结果：

    该基准值会被放置到它最终排序后应该在的位置上，且它左边的所有元素都比它小，右边的所有元素都比它大。

    题目中指出："processing every element which is not yet at its final position is called a run"（处理所有尚未在其最终位置上的元素称为一趟）。这意味着每一趟会处理当前所有未排序的子数组。

    第一趟：处理整个数组，确定 1 个基准值的最终位置，并将数组分为左右两个子数组。

    第二趟：同时处理第一趟产生的左右两个子数组。除非某个子数组为空或只有一个元素，否则每个子数组在这一趟后都会再确定至少 1 个基准值的最终位置。

    因此，如果一个数组状态是经历过两趟快排的结果，那么我们可以反推：它必须能够由上述的过程合理生成。

    **逐个选项验证**

    首先，我们将原数组完全升序排序，作为位置对照的参考：

    目标排序态：[2, 5, 12, 16, 28, 32, 60, 72]

    接下来，我们在每个选项中寻找"合法基准值"（即不仅位置与目标排序态一致，且满足左侧元素全小于它，右侧元素全大于它的元素）。

    **选项 A：** 5, 2, 16, 12, 28, 60, 32, 72

    合法基准值：28, 72。

    过程推理：第一趟选取 72 为基准，右侧子数组为空，左侧为剩下所有元素。第二趟处理左侧子数组，选取 28 为基准。这会产生当前的数组状态。A 是可能的。

    **选项 B：** 2, 16, 5, 28, 12, 60, 32, 72

    合法基准值：2, 72。

    过程推理：第一趟选取 72 为基准，左侧为剩下所有元素。第二趟处理左侧子数组，选取 2 为基准。这会产生当前的数组状态。B 是可能的。

    **选项 C：** 2, 12, 16, 5, 28, 32, 72, 60

    合法基准值：2, 28, 32。

    过程推理：第一趟选取 28 为基准，将数组分为左侧 [2, 12, 16, 5] 和右侧 [32, 72, 60]。第二趟必须处理这两个子数组：左侧选取 2 为基准，右侧选取 32 为基准。这会产生当前的数组状态。C 是可能的。

    **选项 D：** 5, 2, 12, 28, 16, 32, 72, 60

    合法基准值：12, 32。

    我们来尝试反推过程：

    如果第一趟的基准是 32，那么它会将数组分成左侧 [5, 2, 12, 28, 16] 和右侧 [72, 60]。在第二趟中，算法必须同时处理这两个子数组。处理右侧的 [72, 60] 后，必然会在这两个数字中产生至少一个确定了最终位置的基准（也就是变成 60, 72）。但在选项 D 中，右侧依然是未处理的 72, 60，这说明第二趟没有处理右侧，违背了题目"处理所有尚未在其最终位置上的元素"的定义。

    同理，如果第一趟的基准是 12，左侧子数组是 [5, 2]。第二趟必须处理它，使其产生至少一个基准，但在选项 D 中左侧依然是倒序的 5, 2，没有任何元素在最终位置。

    结论：D 无法通过完整的两趟快排得到。

### 2-2

Given input { 321, 156, 57, 46, 28, 7, 331, 33, 34, 63 }. Which one of the following is the result after the 2nd run of the Least Signification Digit (LSD) radix sort?

A. →7→321→28→34→33→331→156→46→57→63

B. →7→321→28→331→34→33→46→156→57→63

C. →156→28→321→331→33→34→46→57→63→7

D. →7→321→28→331→33→34→46→156→57→63

??? success "答案与解析"
    **答案：D**

    LSD（Least Significant Digit）基数排序按位从低到高进行稳定排序。

    **第 1 趟（按个位排序）：**

    个位数分布：
    | 个位 | 元素 |
    |------|------|
    | 1 | 321, 331 |
    | 3 | 33, 63 |
    | 4 | 34 |
    | 6 | 156, 46 |
    | 7 | 57, 7 |
    | 8 | 28 |

    排序后：321, 331, 33, 63, 34, 156, 46, 57, 7, 28

    **第 2 趟（按十位排序）：**

    对上一趟结果按十位稳定排序：

    | 十位 | 元素 |
    |------|------|
    | 0 (007) | 7 |
    | 2 | 321, 28 |
    | 3 | 331, 33, 34 |
    | 4 | 46 |
    | 5 | 156, 57 |
    | 6 | 63 |

    排序后：**7, 321, 28, 331, 33, 34, 46, 156, 57, 63**

    对应选项 D：→7→321→28→331→33→34→46→156→57→63

### 2-3

When running internal sorting, if merge sort is chosen instead of insertion sort, the possible reason should be:

1. The code of merge sort is shorter
2. Merge sort takes less space
3. Merge sort runs faster

A. 2 only

B. 3 only

C. 1 and 2

D. 1 and 3

??? success "答案与解析"
    **答案：B**

    逐一分析三个理由：

    1. **代码更短**——错误。插入排序的实现只有几行，归并排序的递归 + merge 代码显然更长。
    2. **占用空间更少**——错误。插入排序是原地排序，额外空间 O(1)；归并排序需要 O(N) 的辅助数组。
    3. **运行更快**——正确。插入排序平均 O(N²)，归并排序稳定 O(N log N)，数据量大时归并快得多。

    因此只有理由 3 成立。

### 2-4

Among the following sorting methods, which ones will be slowed down if we store the elements in a linked structure instead of a sequential structure?

1. Insertion sort; 
2. Selection Sort; 
3. Bubble sort; 
4. Shell sort; 
5. Heap sort

A. 1 and 2 only

B. 2 and 3 only

C. 3 and 4 only

D. 4 and 5 only

??? success "答案与解析"
    **答案：D**

    判断依据：链表结构不支持 O(1) 随机访问，依赖随机访问的排序算法会被严重拖慢。

    - **Insertion Sort**——可以用链表实现，插入本身 O(1)，找位置遍历 O(N)，总复杂度仍是 O(N²)，不会变慢。
    - **Selection Sort**——每轮遍历找最小元素，链表遍历即可，O(N²)，不受影响。
    - **Bubble sort**——相邻比较交换，链表顺序遍历即可，O(N²)，不受影响。
    - **Shell sort**——❌ 需要按 gap 跳跃访问元素，链表每次跳跃都要遍历 O(gap)，性能大幅下降。
    - **Heap sort**——❌ 需要数组下标的 O(1) 随机访问来维护堆（sift-down/sift-up），链表无法高效支持。

    因此只有 4 和 5 会变慢。

### 2-5

When selecting a sorting algorithm, which of the following factors must be taken into consideration besides the time and space complexities?

- I、the size of input data
- II、the structure used to store the data
- III、the stability of the algorithm
- IV、the initial condition of the data

A. III only

B. I and II only

C. II, III and IV only

D. I, II, III and IV

??? success "答案与解析"
    **答案：D**

    I **数据规模**——小数据插入排序可能比快排更好，大数据必须考虑渐进复杂度，必须考虑。

    II **存储结构**——数组还是链表会极大影响算法选型（如 2-4 所示），必须考虑。

    III **稳定性**——需要稳定排序的场景（如按多关键字排序），必须考虑。

    IV **初始状态**——数据是否接近有序会影响插入排序（O(N) vs O(N²)）和快速排序（退化风险），必须考虑。

    以上四项在实际选型中都是不可忽略的因素，故 I、II、III、IV 全选。
