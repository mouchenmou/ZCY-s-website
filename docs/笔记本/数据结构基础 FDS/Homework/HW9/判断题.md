### 1-1

**题目：** In a weighted undirected graph, if the length of the shortest path from b to a is 12, and there exists an edge of weight 2 between c and b, then the length of the shortest path from c to a must be no less than 10.

**答案：T（正确）**

**分析：**


已知：`b` 到 `a` 的最短路径长度是 12。`c` 和 `b` 之间有一条权重为 2 的边。

假设 `c` 到 `a` 的最短路径长度小于 10（比如是 9）。

那么我们可以构造一条从 `b` 到 `a` 的新路径：
1. 先从 `b` 走到 `c`（走那条权重为 2 的边）
2. 再从 `c` 走到 `a`（走假设的最短路径，长度 9）

这条新路径的总长度 = $2 + 9 = 11$。

**矛盾出现了：** 我们找到了一条从 `b` 到 `a` 长度为 11 的路，但题目说 `b` 到 `a` 的最短路径是 12。最短路径怎么可能被比它更短的路超过呢？

所以假设不成立。`c` 到 `a` 的最短路径长度 **不可能小于 10** 。命题正确。

### 1-2

**题目：** Let P be the shortest path from S to T. If the weight of every edge in the graph is incremented by 2, P will still be the shortest path from S to T.

**答案：F（错误）**

**分析：**

给每条边都加 2，**边数不同的路径受到的影响不同**。

**反例：**
```
        S ──(1)── A ──(1)── T      （路径1：2条边，总权重 2）
         ╲                     ╱
          ──────(3)───────────       （路径2：1条边，总权重 3）
```

- 加 2 之前：路径1 总权重 = 2，路径2 总权重 = 3，**路径1 更短**。
- 每条边加 2 后：路径1 总权重 = $2 + 2 \times 2 = 6$，路径2 总权重 = $3 + 2 \times 1 = 5$，**路径2 更短**。

**结论：** 边数更多的路径在每条边都加相同权重时会"吃亏更多"，所以最短路径可能改变。

### 1-3

**题目：** A segment tree can be used to find the greatest common divisor (GCD) of any index range.

**答案：T（正确）**

**分析：**

线段树适用于 **任何具有"区间可合并性"的操作** 。

什么叫区间可合并性？就是：**两个相邻区间的结果，可以合并成更大区间的结果。**

GCD 满足这个性质（结合律）：

$$\gcd(a_1, a_2, ..., a_k, a_{k+1}, ..., a_n) = \gcd(\gcd(a_1, ..., a_k), \gcd(a_{k+1}, ..., a_n))$$

所以可以用线段树来做区间 GCD 查询。


### 为什么 GCD 可以分段算再合并？

**核心原因：GCD 的本质是"取公共因子的交集"。**


#### 例子 1：每层 GCD 值一样的情况

```
数组：[12, 18, 24, 30]
```

```
                    gcd = 6
              ───────┴───────┐
          gcd = 6          gcd = 6
         ┌───┴───┐        ┌───┴───┐
        12      18        24      30
```

- 左边：gcd(12, 18) = 6
- 右边：gcd(24, 30) = 6
- 合并：gcd(6, 6) = 6

验证：gcd(12, 18, 24, 30) = 6 ✓


#### 例子 2：每层 GCD 值不一样的情况

```
数组：[12, 18, 20, 15]
```

```
                    gcd = 1
              ┌───────┴───────┐
          gcd = 6          gcd = 5
         ┌───┴───┐        ┌──────┐
        12      18        20      15
```

**左边组：gcd(12, 18) = 6**
- 12 = 2² × 3
- 18 = 2 × 3²
- 公共因子：2¹ × 3¹ = **6**

**右边组：gcd(20, 15) = 5**
- 20 = 2² × 5
- 15 = 3 × 5
- 公共因子：5¹ = **5**

**再合并：gcd(6, 5) = 1**
- 6 = 2 × 3，5 = 5
- 没有公共因子（互质），所以 = **1**

**验证：直接从四个数一起算**
```
12 = 2² × 3¹
18 = 2¹ × 3²
20 = 2² × 5¹
15 = 3¹ × 5¹
```
- 2 的最小幂次：min(2, 1, 2, 0) = 0（15 没有 2）
- 3 的最小幂次：min(1, 2, 0, 1) = 0（20 没有 3）
- 5 的最小幂次：min(0, 0, 1, 1) = 0（12 和 18 没有 5）

所以 gcd = 2⁰ × 3⁰ × 5 = **1**。**和分段算的结果完全一样。**


#### 为什么"不一样"也能合并？

用集合的语言来说：

- 左边组的公共因子集合：D(12, 18) = {1, 2, 3, 6}
- 右边组的公共因子集合：D(20, 15) = {1, 5}
- 四数一起的公共因子集合：D(12, 18, 20, 15) = {1, 2, 3, 6} ∩ {1, 5} = {1}

**公共因子集合 = 两个子集合的交集。** GCD 就是这个交集中最大的那个。

换句话说：**"左右两组各自的公共因子，再取公共因子" = "四组一起的公共因子"。** 这不是巧合，而是"取交集"这个操作天然可以任意分组。

---

#### 对比：为什么平均值不能这样分？

```
[3, 7] 的平均 = 5
[2, 5, 1] 的平均 = 8/3
```

两组平均值合起来算不出整体平均值，因为**平均值不是"交集"关系**——它依赖每组的元素个数（左边 2 个，右边 3 个）。而 GCD 不关心"每组有几个数"，只关心"哪些因子是公共的"。

---

### 总结

| | GCD | 平均值 |
|---|-----|-------|
| 本质 | 公共因子的交集 | 总和除以个数 |
| 分组算再合并 | ✓ 公共因子集合 = 子集交集 | ✗ 需要知道每组有多少个元素 |
| 依赖信息 | 只看因子，不看个数 | 既要看值，也要看个数 |

一句话记住：**只要两个子区间的结果能合并成大区间的结果，就可以用线段树。**

---

**线段树还能干什么？**

| 操作 | 能否用线段树 | 合并方式 |
|------|-------------|----------|
| 区间求和 | ✓ | `left.sum + right.sum` |
| 区间最大/最小 | ✓ | `max(left, right)` / `min(left, right)` |
| 区间 GCD | ✓ | `gcd(left, right)` |
| 区间 XOR | ✓ | `left.xor  right.xor` |
| 区间平均值 | ✗ | 不满足直接可合并性 |
| 区间中位数 |  | 同上 |

时间复杂度：构建 $O(n)$，查询 $O(\log n)$。
