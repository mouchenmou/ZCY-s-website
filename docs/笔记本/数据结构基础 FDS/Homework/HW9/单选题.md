### 2-1

**题目：** What kind of problems can Dijkstra algorithm solve?

**答案：B. shortest path**

**分析：**

Dijkstra 算法用于求解**单源最短路径**问题（带权图，边权重非负）。

- A. critical path（关键路径）→ 用拓扑排序 + 动态规划，不是 Dijkstra
- C. topological sort（拓扑排序）→ 用 DFS 或 Kahn 算法
- D. string matching（字符串匹配）→ 用 KMP、Boyer-Moore 等

### 2-2

**题目：** Use Dijkstra algorithm to find the shortest paths from 1 to every other vertices. In which order that the destinations must be obtained?

![](附件/Pasted%20image%2020260511215257.png)

**答案：B. 2, 4, 3, 6, 5, 7**

**分析：**

按 Dijkstra 算法一步步执行：

**初始化：** dist[1]=0，其余为 ∞

| 步骤 | 选取的顶点 | dist 更新 |
|------|-----------|----------|
| 1 | 选 1 (dist=0) | 1→2: dist[2]=5；1→5: dist[5]=35 |
| 2 | 选 2 (dist=5，最小) | 2→4: dist[4]=7；2→3: dist[3]=13；2→6: dist[6]=17 |
| 3 | 选 4 (dist=7，最小) | 4→3: dist[3]=11（更新！）；4→7: dist[7]=22 |
| 4 | 选 3 (dist=11，最小) | 3→7: dist[7]=12（更新！） |
| 5 | 选 7 (dist=12) | 7→6: dist[6]=15（更新！） |
| 6 | 选 6 (dist=15) | 6→5: dist[5]=16（更新！） |
| 7 | 选 5 (dist=16) | 结束 |

**确定顺序：2 → 4 → 3 → 7 → 6 → 5**

但选项中没有完全匹配的。前三个顶点 2, 4, 3 的顺序是确定无疑的，选项中只有 B 的前三个是 2, 4, 3。

!!! warning "注意"
    这道题的关键在于前三个顶点的顺序：2（距离5）→ 4（距离7）→ 3（距离11），这是由图的边权重决定的，不可能出错。选项中只有 B 的前三个是 2, 4, 3。

### 2-3

**题目：** If besides finding the shortest path from S to every other vertices, we also need to count the number of different shortest paths, we can modify the Dijkstra algorithm in the following way: add an array count[] so that count[V] records the number of different shortest paths from S to V. Then count[V] shall be initialized as:

**答案：A. count[S]=1; and count[V]=0 for other V**

**分析：**

**原理：**源点 S 到自己的最短路径只有一条（长度为 0 的"空路径"），所以 count[S]=1。其余顶点一开始都不可达，所以 count[V]=0。

在 Dijkstra 的松弛过程中，需要额外维护 count 数组：

- 如果 `dist[u] + w < dist[v]`（发现更短路径）：
  - `dist[v] = dist[u] + w`
  - `count[v] = count[u]`（继承 u 的最短路径条数）

- 如果 `dist[u] + w == dist[v]`（发现同样短的另一条路径）：
  - `count[v] += count[u]`（累加路径条数）

### 2-4

**题目：** When Dijkstra's algorithm is used to find the shortest paths from v₁ to every other vertices in an undirected graph G, a distance set dist[v] is maintained for every vertex v... Suppose that the destinations are obtained in the order of {v₂, v₃, ⋯, vₙ}. If there is a vertex w in G so that dist[w] is decreased during every iteration (except the last one) of the algorithm, how many statements of the following are correct?

(1) w is adjacent to every other vertices.
(2) (v₁, w) (if it exists) is the longest edge among all the edges that are adjacent to w.
(3) Such a vertex w does not exist.

**答案：C. 2**

**分析：**

**（1）w 必须与所有其他顶点相邻。** ✓

要让 dist[w] 在**每一次**迭代都被更新，意味着**每一个被出队的顶点 vᵢ 都必须有边连到 w**，否则处理 vᵢ 时 dist[w] 不会被更新。所以 w 必须与 v₂, v₃, , vₙ 都相邻。**（1）正确。**

**（2）(v₁, w) 是最长的边。** ✓

这是核心推导。

初始时，dist[w] 的初始值为 w(v₁, w)（如果边存在）。第一轮处理 v₂ 后：

$$dist[w] = dist[v₂] + w(v₂, w) < w(v₁, w)$$

注意这里**dist[v₂] ≥ 0**（Dijkstra 边权非负），所以：

$$w(v₂, w) < w(v₁, w)$$

第二轮处理 v₃ 时，要继续更新：

$$dist[v] + w(v₃, w) < dist[w]_{\text{当前}} < w(v₁, w)$$

同样因为 dist[v₃] ≥ 0，必有：

$$w(v₃, w) < w(v₁, w)$$

**以此类推，每一轮都要求 w(v, w) < w(v₁, w)。**所以 (v₁, w) 必须是 w 的所有邻边中最长的。**（2）正确。**

!!! warning "我之前的错误反例"
    我之前构造的反例：dist[v₂]=1, dist[v₃]=50, dist[v₄]=10，边权 w(v₂,w)=0.5, w(v₃,w)=2, w(v₄,w)=2。
    
    第一轮后 dist[w] = 1 + 0.5 = 1.5。
    
    第二轮 v₃ 过来：dist[v] + w(v₃,w) = 50 + 2 = 52 > 1.5，**根本不会更新！**
    
    问题在于我选的 dist[vᵢ] 太大了，后续的路径长度根本追不上已经降下来的 dist[w]。正确的构造需要 w(vᵢ,w) 随着 dist[vᵢ] 增大而急剧减小，这非常受限，但并非不可能（例如 w(v,w) 指数衰减）。

**（3）这样的顶点 w 不存在。** ✗

w 是**存在的**，只要满足以下条件即可：

- w(v₁, w) 足够大
- w(vᵢ, w) 随着 dist[v] 增大而快速减小

**可行构造示例：**

设置：dist[v₂]=1, dist[v₃]=2, dist[v₄]=3
边权：w(v₁,w)=1000, w(v₂,w)=100, w(v₃,w)=50, w(v₄,w)=10

验证每轮更新：
- 初始：dist[w] = 1000
- 处理 v₂：1 + 100 = 101 < 1000 ✓ 更新
- 处理 v₃：2 + 50 = 52 < 101 ✓ 更新
- 处理 v₄：3 + 10 = 13 < 52 ✓ 更新

**dist[w] 每轮都下降了，w 确实存在。所以（3）错误。**

**综上，（1）和（2）正确，选 C。**

### 2-5

**题目：** What is a segment tree primarily used for?

**答案：B. Finding minimum/maximum in a range**

**分析：**

线段树适用于**任何具有"区间可合并性"的操作**。

什么叫区间可合并性？就是：**两个相邻区间的结果，可以合并成更大区间的结果。**

**老师上课举例的区间求和**就是最典型的例子：

```
数组：[3, 7, 2, 5, 1, 8, 4, 6]
线段树：
              [0-7] = 36
        ─────────────┴─────────────┐
    [0-3] = 17                  [4-7] = 19
    ┌───┴───┐                   ┌───┴───┐
 [0-1]=10 [2-3]=7           [4-5]=9  [6-7]=10
  ┌──┐   ┌─┴─              ┌─┴─   ┌─┴─┐
  3   7   2   5              1   8   4   6
```

查 [1,5] 的和：拆成 [1,1] + [2,3] + [4,5] = 7 + 7 + 9 = **23**

合并操作就是**加法**：`left.sum + right.sum`。

**区间最值（本题的正确答案）**原理完全一样：

```
数组：[3, 7, 2, 5, 1, 8, 4, 6]
线段树每个节点存对应区间的最大值：
              max = 8
        ───────┴───────┐
     max = 7         max = 8
    ┌──────┐        ───┴───┐
   7      5         8      6
```

查 [1,5] 的最大值：拆成 [1,1]、[2,3]、[4,5] → 7、5、8 → **max(7, 5, 8) = 8**

合并操作就是**取 max**：`max(left.max, right.max)`。

**选项分析：**

- **A. Sorting** → 线段树不是排序算法
- **B. Finding minimum/maximum in a range** → ✓ 最典型应用
- **C. Finding the sum** → 虽然也能做区间求和，但题目说"a given set"（给定集合）而不是"a range"（区间），且区间最值是线段树"primarily"的用途
- **D. String matching** → 线段树不用于字符串匹配

---

**一句话记住线段树的本质：**

> 只要两个子区间的结果能合并成大区间的结果（满足结合律），就可以用线段树。

**常见的可合并操作：**求和、最值、GCD、XOR

**不能直接用线段树的操作：**平均值、中位数（不满足直接可合并性）
