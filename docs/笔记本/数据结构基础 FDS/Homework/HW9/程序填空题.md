### 概念解释

#### NodePtr 是什么？

`NodePtr` 是一个**边节点的指针类型**，用来遍历邻接表。

在这道题的代码里，图是用**邻接表**（Adjacency List）存储的。每个顶点都有一个链表，链表的每个节点代表一条从该顶点出发的边。

```
G->List[V].FirstEdge → [边1] → [边2] → [边3] → NULL
                            ↓      ↓      ↓
                         AdjV=2 AdjV=5 AdjV=7
```

看代码第 10 行：

```c
for ( ptr=G->List[V].FirstEdge; ptr; ptr=ptr->Next )
```

- `ptr` 从 `V` 的第一条边开始
- 每次 `ptr = ptr->Next` 走到下一条边
- 走到 `ptr` 为 NULL 时停止（即 `ptr` 为假）

代码第 11 行：

```c
U = ptr->AdjV;
```

- `ptr->AdjV` 取当前这条边**指向的顶点编号**

**一句话：** `NodePtr ptr` 就是**"当前正在看的那条边"的指针**，顺着它就能把顶点 V 的所有邻居都走一遍。

#### Enqueue 是什么？

`Enqueue(S, Q)` 就是**把顶点 S 放入队列 Q**。

这道题用的是 BFS（广度优先搜索），BFS 的核心数据结构就是**队列**。流程是：

1. 起点 S 入队：`Enqueue(S, Q)`
2. 循环：出队一个顶点 V → 遍历 V 的所有邻居 → 没访问过的邻居入队

所以 `Enqueue` 就是 BFS 里的"入队"操作，和你猜的一样。
