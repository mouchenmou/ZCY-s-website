### 1. DFS on DAG → Output Before Recursion Ends

> Apply DFS to a directed acyclic graph, and output the vertex **before the end of each recursion**. The output sequence will be:

- A. unsorted
- B. topologically sorted
- C. reversely topologically sorted
- D. None of the above

??? success "答案与解析"
    **答案：C**
    
    在 DFS 中，"before the end of each recursion"（递归返回前输出）对应的是后序遍历（post-order）。
    
    在有向无环图（DAG）中：后序遍历序列的逆序 = 拓扑排序。因此递归返回前的输出序列是逆拓扑序。
    
    !!! tip "记忆技巧"
        - 递归进入时输出 → pre-order
        - 递归返回前输出 → post-order → 逆序即拓扑排序

### 2. Euler Circuit in K₂₀

> Graph G is an undirected **complete graph of 20 nodes**. Is there an Euler circuit in G? If not, to have an Euler circuit, what is the minimum number of edges which should be removed?

- A. Yes, Graph G has an Euler circuit
- B. No, Graph G has no Euler circuit. 10 edges should be removed.
- C. No, Graph G has no Euler circuit. 20 edges should be removed.
- D. No, Graph G has no Euler circuit. 40 edges should be removed.

??? success "答案与解析"
    **答案：B**
    
    **Step 1 — 判定有没有欧拉回路：**
    
    完全图 $K_{20}$ 中，每个顶点与其余 19 个顶点相连 → **每个顶点度数 = 19（奇数）**。
    
    无向图存在欧拉回路 $\iff$ **所有顶点度数为偶数**。$K_{20}$ 不满足，所以**没有欧拉回路**。
    
    **Step 2 — 最少删几条边：**
    
    - 每条边被删除，会让两个端点的度数各减 1（奇偶性翻转）
    - 当前 20 个顶点全是奇数度 → 需要把所有奇度变成偶度
    - 删一条边翻转两个顶点的奇偶性 → 至少需要**$20 \div 2 = 10$**条边
    
    具体构造：把 20 个顶点分成 10 对，删掉每对之间的边即可。

### 3. Impossible DFS Sequence (from Adjacency Matrix)

> Given the adjacency matrix of a graph as shown by the figure. Then starting from V5, an impossible DFS sequence is:

![](附件/Pasted%20image%2020260526201328.png)

- A. V5, V6, V3, V1, V2, V4
- B. V5, V1, V2, V4, V3, V6
- C. V5, V1, V3, V6, V4, V2
- D. V5, V3, V1, V2, V4, V6

??? success "答案与解析"
    **答案：C**
    
    图的邻接关系：1—2, 1—3, 1—5, 2—4, 3—5, 3—6, 4—5, 5—6
    
    **逐个验证：**
    
    | 选项 | 序列 | 判定 |
    |------|------|------|
    | A | V5→V6→V3→V1→V2→V4 | ✓ V5 先走 V6，一路深入到底 |
    | B | V5→V1→V2→V4→V3→V6 | ✓ V5→V1→V2→V4 到底，回溯 V1→V3→V6 |
    | **C**|**V5→V1→V3→V6→V4→V2**|**✗ V4 不可能排在 V2 前面** |
    | D | V5→V3→V1→V2→V4→V6 | ✓ V5→V3→V1→V2→V4，回溯 V3→V6 |
    
    **C 为什么不可能？**
    
    V5→V1→V3→V6 之后，V6 没有未访问邻居 → 回溯到 V3（也没有）→ 回溯到 V1。V1 只剩未访问邻居**V2**。V4 不是 V1 的邻居，只能通过 V2 到达。所以下一步**必须是 V2**，V4 不可能直接出现在 V6 之后。

### 4. Insertion Sort: Comparisons and Movements

> Use simple insertion sort to sort 10 numbers **from non-decreasing to non-increasing**, the possible numbers of comparisons and movements are:

- A. 100, 100
- B. 100, 54
- C. 54, 63
- D. 45, 44

??? success "答案与解析"
    **答案：D**
    
    "from non-decreasing to non-increasing" = 输入递增，要排成递减 → **最坏情况**。
    
    | P | 比较 | 移动 |
    |---|------|------|
    | 1 | 1 | 1 |
    | 2 | 2 | 2 |
    | $\vdots$ | $\vdots$ | $\vdots$ |
    | 9 | 9 | 9 |
    | **总计**|**45**|**44** |
    
    $$\text{比较次数} = \frac{N(N-1)}{2} = 45$$
    
    $$\text{移动次数} = \frac{N(N-1)}{2} - 1 = 44$$
    
    移动少一次是因为第一趟 $P=1$ 时把 Tmp 取出的那一次不计入"移动"。

### 5. Binary Insertion Sort: Max Comparisons

> Given a **sorted file of 1000 records**. To insert a new record by insertion sort with **binary search**, the maximum number of comparisons is:

- A. 1000
- B. 999
- C. 500
- D. 10

??? success "答案与解析"
    **答案：D**
    
    折半插入排序用**二分查找**定位插入位置，比较次数取决于二分查找的深度，而非数组大小。
    
    在 1000 个有序记录中二分查找：
    
    $$\lceil \log_2 1000 \rceil = 10 \quad (\text{因为 } 2^9 = 512 < 1000 \leq 1024 = 2^{10})$$
    
    所以最多比较**10 次**。
    
    !!! warning "注意"
        这里问的是**比较次数**，不是移动次数。二分查找只减少了比较次数（从 $O(N)$ 降到 $O(\log N)$），元素的移动次数仍然是 $O(N)$。

### 6. Insertion Sort: Fewest Comparisons

> Using the direct insertion sort algorithm to sort the following four lists in ascending order, the one with the **fewest number of comparisons** is:

- A. {134, 26, 40, 110, 80, 54, 12, 62}
- B. {12, 26, 54, 40, 80, 62, 110, 134}
- C. {26, 40, 12, 54, 62, 134, 110, 80}
- D. {110, 62, 80, 54, 12, 26, 134, 40}

??? success "答案与解析"
    **答案：B**
    
    插入排序的比较次数 = $\sum$ (每趟停留在正确位置前需要的比较次数)。数组**越接近有序**，比较次数越少。
    
    计算每个选项的**逆序对数量**（逆序对越少 $\to$ 越有序）：
    
    | 选项 | 数组 | 逆序对数 |
    |------|------|----------|
    | A | {134, 26, 40, 110, 80, 54, 12, 62} | 16 |
    | **B**|**{12, 26, 54, 40, 80, 62, 110, 134}**|**2** ← 最少 |
    | C | {26, 40, 12, 54, 62, 134, 110, 80} | 5 |
    | D | {110, 62, 80, 54, 12, 26, 134, 40} | 18 |
    
    **B 仅有 2 个逆序对**（54→40 和 80→62），几乎有序，因此比较次数最少。
    
    !!! tip "速判法"
        不用一个个数逆序对——扫一眼哪个数组最接近有序就行。B 只需要交换两处（54↔40、80↔62）就完全有序了。
