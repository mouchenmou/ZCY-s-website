## 1. Bellman-Ford 算法

### 1.1 算法思想

Bellman-Ford 算法解决的是 **带负权边的单源最短路** 问题。

Dijkstra 算法之所以不能处理负权边，是因为它的贪心策略（每次选距离最小的顶点加入已处理集合 $S$）在负权边存在时可能出错。Bellman-Ford 放弃了贪心，改用一种更"笨"但更通用的方法：**松弛操作**。

### 1.2 算法具体实现

设图 $G = (V, E)$，顶点数 $|V| = n$，边数 $|E| = m$。源点为 $s$。

```c
void BellmanFord(Graph G, Vertex s, Table T) {
    // 初始化
    for (each vertex V) {
        T[V].Dist = Infinity;
        T[V].Path = NotAVertex;
    }
    T[s].Dist = 0;
    
    // 核心：重复 |V|-1 轮松弛
    for (int i = 1; i <= |V| - 1; i++) {
        for (each edge (u, v) with weight w in E) {
            if (T[u].Dist != Infinity && T[u].Dist + w < T[v].Dist) {
                T[v].Dist = T[u].Dist + w;
                T[v].Path = u;
            }
        }
    }
    
    // 检测负环
    for (each edge (u, v) with weight w in E) {
        if (T[u].Dist != Infinity && T[u].Dist + w < T[v].Dist) {
            Error("Graph contains negative cycle");
        }
    }
}
```

**算法流程：**

1. **初始化**：源点距离为 0，其余为 $\infty$
2. **$|V|-1$ 轮松弛**：每轮遍历所有边，尝试用 $u$ 去更新 $v$ 的距离
3. **额外检测**：再做一轮，若还能更新 $\Rightarrow$ 存在负环

### 1.3 算法的正确性证明

#### 定理：如果图中没有负环，Bellman-Ford 执行 $|V|-1$ 轮松弛后，所有顶点的最短距离都已确定。

**证明：**

我们要证明：对于任意顶点 $v$，如果从源点 $s$ 到 $v$ 存在一条最短路径，那么执行 $|V|-1$ 轮松弛后，`T[v].Dist` 一定等于该最短路径的长度。

设 $s$ 到 $v$ 的最短路径为 $s \to v_1 \to v_2 \to \cdots \to v_k \to v$，这条路径上有 $k$ 条边，$k+1$ 个顶点（不含 $s$）。由于图中没有负环，最短路径一定是 **简单路径**（不含环），所以 $k \le |V|-1$。

我们用 **数学归纳法** 证明：

**归纳命题 $P(i)$**：第 $i$ 轮松弛后，所有距离源点最短路径 $\le i$ 条边的顶点，其 `Dist` 值已收敛到正确的最短距离。

- **基础：$i = 0$**（初始化后）
    - 源点 $s$ 的最短路径有 0 条边，`T[s].Dist = 0`，正确。
    - 其余顶点最短路径 $\ge 1$ 条边，`Dist = \infty`，暂未确定。

- **归纳：假设 $P(i)$ 成立**，证明 $P(i+1)$ 也成立。
    - 设顶点 $v$ 从 $s$ 到 $v$ 的最短路径有 $i+1$ 条边：$s \to v_1 \to v_2 \to \cdots \to v_i \to v$。
    - 这条路径的前 $i$ 条边构成的子路径 $s \to v_1 \to \cdots \to v_i$ 也是最短路径（最短路径的子路径也是最短路径），且恰好 $i$ 条边。
    - 由归纳假设 $P(i)$，第 $i$ 轮后 `T[v_i].Dist` 已经等于 $s \to v_i$ 的正确最短距离。
    - 第 $i+1$ 轮松弛时，边 $(v_i, v)$ 会被扫描到，此时：
        $$T[v_i].Dist + w(v_i, v) = \text{正确最短距离}$$
        所以 `T[v].Dist` 会被更新为正确的最短距离。
    - 证毕。

**结论**：第 $|V|-1$ 轮后，所有顶点的最短距离都已正确计算。

---

### 1.4 为什么执行 $|V|-1$ 轮后若还能更新就说明有负环？

**证明（反证法）：**

假设图中 **没有负环**，但执行 $|V|-1$ 轮后第 $|V|$ 轮仍能更新某条边 $(u, v)$ 的距离，即：

$$T[u].Dist + w(u, v) < T[v].Dist$$

这说明 `T[v].Dist` 在第 $|V|-1$ 轮后还没有收敛到正确的最短距离。

但根据上面的定理，如果图中没有负环，$|V|-1$ 轮后所有顶点的最短距离都应该已确定。这意味着 $s \to v$ 的最短路径的边数 $\ge |V|$。

然而，$s \to v$ 的路径经过 $|V|$ 条边意味着路径上有 $|V|+1$ 个顶点（含 $s$ 和 $v$）。但图中总共只有 $|V|$ 个顶点，所以这条路径上 **一定有重复的顶点**，即存在环。

又因为 $s \to v$ 是"最短路径"，说明这个环的总权重 **不能是正数**（否则去掉这个环路径会更短），也不能是 0（去掉也无损），所以这个环的总权重一定是 **负数**。

这与"假设图中没有负环"矛盾！

**因此：如果 $|V|-1$ 轮后还能更新，图中一定存在负环。**

---

### 1.5 负环为什么导致没有最短路径？

如果图中存在负环（从源点可达），那么可以无限次绕这个负环走，每次绕一圈路径总权重就减少，永远不存在一个"最小"的值。也就是说，**最短路径的值趋于 $-\infty$，不存在有限的最短路径**。

## 2. SPFA 算法（Shortest Path Faster Algorithm）

### 2.1 算法思想

SPFA 是对 Bellman-Ford 的优化。Bellman-Ford 每一轮都 **无差别地扫描所有边**，但很多边的端点距离根本没有变化，这些松弛操作是浪费的。

**核心观察：** 只有当 $u$ 的 `Dist` 被更新后，$u$ 的出边才有可能去更新邻居的距离。所以可以用队列来 **只处理发生了更新的顶点**。

### 2.2 算法具体实现

```c
void SPFA(Graph G, Vertex s, Table T) {
    Queue Q = CreateQueue();
    int count[|V|] = {0};  // 记录每个顶点入队次数
    bool inQueue[|V|] = {false};  // 标记顶点是否在队列中
    
    for (each vertex V) {
        T[V].Dist = Infinity;
        T[V].Path = NotAVertex;
    }
    T[s].Dist = 0;
    
    Enqueue(s, Q);
    inQueue[s] = true;
    
    while (!IsEmpty(Q)) {
        u = Dequeue(Q);
        inQueue[u] = false;
        
        for (each edge (u, v) with weight w) {
            if (T[u].Dist + w < T[v].Dist) {
                T[v].Dist = T[u].Dist + w;
                T[v].Path = u;
                
                if (!inQueue[v]) {
                    count[v]++;
                    if (count[v] >= |V|) {
                        Error("Graph contains negative cycle");
                    }
                    Enqueue(v, Q);
                    inQueue[v] = true;
                }
            }
        }
    }
}
```

### 2.3 与 Bellman-Ford 的关系

- SPFA 是 Bellman-Ford 的 **队列优化版本**
- 每一轮只松弛"距离发生了变化"的顶点
- 最坏情况（构造特殊图）下，时间复杂度仍为 $O(|V| \cdot |E|)$，与 Bellman-Ford 相同
- 但 **平均情况** 下快得多，通常 $O(k \cdot |E|)$，其中 $k$ 是一个很小的常数（一般 $k < 2$）

### 2.4 为什么某个顶点更新超过 $|V|-1$ 次就说明有负环？

**证明：**

先明确：这里的"更新"指的是 `T[v].Dist` 被成功松弛（值变小），"入队次数"指的是顶点被加入队列的次数。我们证明两者在负环检测中等价。

**关键观察：** 如果图中没有负环，任何顶点的最短路径一定是简单路径，最多有 $|V|-1$ 条边，因此该顶点的 `Dist` **最多被更新 $|V|-1$ 次**（从 $\infty$ 更新到某个有限值，最多 $|V|-1$ 次逐步逼近）。

更严谨地说：

1. **Bellman-Ford 框架下的理解**：
    - 在 Bellman-Ford 中，第 $i$ 轮松弛后，所有最短路径 $\le i$ 条边的顶点都已确定正确距离
    - 如果某个顶点 $v$ 的最短路径恰好需要 $k$ 条边，那么它的 `Dist` 恰好会被更新 $k$ 次（在第 $1, 2, \ldots, k$ 轮中各被更新一次）
    - 由于没有负环时 $k \le |V|-1$，所以任何顶点的 `Dist` 最多被更新 $|V|-1$ 次

2. **SPFA 中用入队次数来判断**：
    - SPFA 中，顶点 $v$ 每次被松弛就会入队一次
    - 如果图中没有负环，$v$ 的最短路径最多有 $|V|-1$ 条边，所以 $v$ 最多被松弛 $|V|-1$ 次
    - 如果 $v$ 入队次数 $\ge |V|$，说明 $v$ 的 `Dist` 被更新了至少 $|V|$ 次
    - 这意味着从 $s$ 到 $v$ 的最短路径有至少 $|V|$ 条边，路径上有至少 $|V|+1$ 个顶点
    - 图中只有 $|V|$ 个顶点 $\Rightarrow$ 路径上一定有重复顶点 $\Rightarrow$ 存在环
    - 而这条路径还能继续被松弛 $\Rightarrow$ 环的权重为负
    - **所以存在负环**

### 2.5 SPFA 负环检测的完整逻辑

在 SPFA 的实际代码中，判断负环有两种常用方法：

**方法 1：统计入队次数**
- 维护一个 `count[v]` 数组，记录每个顶点入队的次数
- 如果 `count[v] >= |V|`，则存在负环
- 原理就是上面证明的内容

**方法 2：统计最短路径边数**
- 维护一个 `cnt[v]` 数组，记录当前从 $s$ 到 $v$ 的最短路径包含的边数
- 初始 `cnt[s] = 0`，其余为 0
- 每次松弛 `T[v].Dist = T[u].Dist + w` 时，同时更新 `cnt[v] = cnt[u] + 1`
- 如果 `cnt[v] >= |V|`，则存在负环
- 这个方法的物理意义更直观：如果最短路径的边数 $\ge |V|$，说明路径上有重复顶点，即存在环

两种方法等价，但 **方法 2 更严格**（方法 1 在某些特殊图中可能误判，因为入队 $\ge |V|$ 不一定意味着同一条路径上的环）。

### 2.6 注意事项

- **SPFA 的最坏情况**：可以构造特殊图让 SPFA 退化为 $O(|V| \cdot |E|)$，所以在竞赛中如果题目卡 SPFA 需要用 Bellman-Ford 或 Johnson 算法
- **负环时 SPFA 不会自动终止**：如果没有负环检测，遇到负环会无限循环（每次绕环距离都变小，每次都会入队）
- **实际表现**：在随机图和大部分实际数据中，SPFA 远快于 Dijkstra（因为不需要堆操作），但理论上不如 Dijkstra 稳定
