## 基本概念

### 稳定性（Stable）

!!! conception "稳定排序"
    如果序列中有两个**相等**的元素 $A[i] = A[j]$（$i < j$），排序后它们的**相对位置不变**，则称该排序算法是**稳定的**。

!!! example "例子"
    原始：`(3, a), (1, b), (3, c)`，按数字排序。
    
    - 稳定排序结果：`(1, b), (3, a), (3, c)` — `(3, a)` 仍在 `(3, c)` 前面
    - 不稳定排序结果可能：`(1, b), (3, c), (3, a)` — 两个 3 的相对位置变了

### 原地排序（In-place）

!!! conception "原地排序"
    排序过程中除了输入数组本身外，**只使用 $O(1)$ 的额外空间**（常数个临时变量）。

## 1. Insertion Sort（插入排序）

!!! abstract "类比：摸牌"
    就像打牌时摸牌——你手上已经有一把排好序的牌，摸到一张新牌后，从右往左找它应该在的位置，把比它大的牌都往右挪一位，然后把新牌塞进去。

```c
void InsertionSort(ElementType A[], int N) {
    int j, P;
    ElementType Tmp;

    for (P = 1; P < N; P++) {
        Tmp = A[P];                      /* 摸起下一张牌 */
        for (j = P; j > 0 && A[j - 1] > Tmp; j--)
            A[j] = A[j - 1];             /* 比 Tmp 大的牌往右挪 */
        A[j] = Tmp;                      /* 把新牌放到正确位置 */
    }
}
```

- 进行 $N-1$ 趟排序（第 $P$ 趟时，位置 $0$ 到 $P-1$ 已排好序）
- 第 $P$ 趟把第 $P$ 个元素插入到前面有序序列的正确位置

| 情况 | 复杂度 | 说明 |
|------|--------|------|
| 最坏 | $O(N^2)$ | 输入是**逆序**的，每趟都要把所有已排序元素右移 |
| 最好 | $O(N)$ | 输入已经**有序**，每趟只比较一次就结束 |
| 平均 | $O(N^2)$ | 见下方逆序对分析 |

### 1.1 逆序对（Inversion）与下界分析

!!! conception "逆序对（Inversion）"
    对于数组 $A$，若 $i < j$ 但 $A[i] > A[j]$，则称 $(i, j)$ 是一个**逆序对**。

!!! example "例子"
    数组 `34, 8, 64, 51, 32, 21` 有 **9** 个逆序对：
    
    (34,8) (34,32) (34,21) (64,51) (64,32) (64,21) (51,32) (51,21) (32,21)
    
    插入排序恰恰需要 9 次交换来把这个数组排好序。

**核心观察：** 插入排序每次交换两个相邻的元素，恰好消除恰好一个逆序对。

因此，插入排序的运行时间 = $O(I + N)$，其中 $I$ 是原数组中的逆序对数量。数组越接近有序（$I$ 越小），插入排序越快。

!!! abstract "两个定理"
    - **平均逆序对数量：** $N$ 个不同元素的数组中，平均有 $N(N-1)/4$ 个逆序对。
    - **下界定理：** 任何只交换相邻元素的排序算法，平均时间复杂度为 $\Omega(N^2)$。

!!! tip "为什么这很重要？"
    要想突破 $O(N^2)$，必须做相隔很远的元素之间的比较和交换——而不是只交换相邻元素。
    
    这就是希尔排序（Shell Sort）和后面那些 $O(N\log N)$ 算法的核心动机。

### 1.2 稳定性与原地性

- **Stable?** ✅ 稳定。`A[j-1] > Tmp` 只在严格大于时才移动，相等的元素不会穿越彼此。
- **In-place?** ✅ 原地。只用了一个临时变量 `Tmp`，额外空间 $O(1)$。

## 2. 希尔排序（Shell Sort）

!!! abstract "核心思想"
    插入排序每次只交换相邻元素，一次只消一个逆序对，太慢了。
    
    **希尔排序**：一次消除多个逆序对——先比较相隔很远的元素，逐渐缩小步长，最后 $h=1$ 退化回普通插入排序。

### 2.1 增量序列与过程

**增量序列** $h_1 < h_2 < \cdots < h_t$，其中 $h_1 = 1$。对每个 $h_k$，做一次 $h_k$**-sort**：把数组按间隔 $h_k$ 分组，每组内做插入排序。

!!! example "例子：Shell 增量序列（$h_t = \lfloor N/2 \rfloor, h_k = \lfloor h_{k+1}/2 \rfloor$）"
    ```
    原始: 81  94  11  96  12  35  17  95  28  58  41  75  15
    
    5-sort: 35  17  11  28  12  41  75  15  96  58  81  94  95
    3-sort: 28  12  11  35  15  41  58  17  94  75  81  96  95
    1-sort: 11  12  15  17  28  35  41  58  75  81  94  95  96
    ```

**重要性质：** $h_k$-sorted 的序列在 $h_{k-1}$-sorted 后仍保持 $h_k$-sorted。

### 2.2 代码实现

```c
void Shellsort(ElementType A[], int N) {
    int i, j, Increment;
    ElementType Tmp;
    for (Increment = N / 2; Increment > 0; Increment /= 2)
        for (i = Increment; i < N; i++) {
            Tmp = A[i];
            for (j = i; j >= Increment; j -= Increment)
                if (Tmp < A[j - Increment])
                    A[j] = A[j - Increment];
                else
                    break;
            A[j] = Tmp;
        }
}
```

### 2.3 复杂度分析

| 增量序列 | 最坏 | 平均 |
|----------|------|------|
| Shell（$h_t = \lfloor N/2\rfloor$） | $\Theta(N^2)$ | — |
| Hibbard（$h_k = 2^k-1$） | $\Theta(N^{3/2})$ | $O(N^{5/4})$ |
| Sedgewick（$1, 5, 19, 41, 109, \ldots$） | $O(N^{4/3})$ | $O(N^{7/6})$ |

!!! warning "Shell 增量序列的坏情况"
    当增量不互质时，后面的小增量可能几乎不起作用：
    ```
    1 9 2 10 3 11 4 12 5 13 6 14 7 15 8 16
    → 8-sort, 4-sort, 2-sort 后数组纹丝不动，只有 1-sort 干活
    → 退化到 $O(N^2)$
    ```
    **Hibbard 序列**解决了这个问题——连续增量没有公因子。

### 2.4 稳定性与原地性

- **Stable?** ❌ 不稳定。远距离交换时，相等元素可能被跳过，相对位置被打乱。
- **In-place?** ✅ 原地。额外空间 $O(1)$。

## 3. 堆排序（Heapsort）

### 3.1 算法一：用辅助数组（Naive）

```
1. BuildHeap(H);              // O(N)
2. for i = 0 to N-1:
       TmpH[i] = DeleteMin(H); // O(log N) each
3. for i = 0 to N-1:
       H[i] = TmpH[i];         // O(1) each
```

- 总时间：$O(N \log N)$
- **空间翻倍**：需要额外的 `TmpH[]` 数组

### 3.2 算法二：原地堆排序（In-place）

改用**大顶堆**，堆顶（最大元素）换到末尾，然后下沉。下标 0 作为堆的起点。

```c
void Heapsort(ElementType A[], int N) {
    int i;
    for (i = N / 2; i >= 0; i--)   /* BuildHeap: O(N) */
        PercDown(A, i, N);
    for (i = N - 1; i > 0; i--) {
        Swap(&A[0], &A[i]);         /* DeleteMax */
        PercDown(A, 0, i);
    }
}
```

**复杂度：**
- 建堆：$O(N)$（从 N/2 下沉到 0）
- $N-1$ 次 DeleteMax，每次 $O(\log N)$
- 总时间：$O(N \log N)$

!!! abstract "定理"
    对 $N$ 个不同元素的随机排列做堆排序，平均比较次数为：
    $$2N\log N - O(N\log\log N)$$

!!! warning "实际表现"
    虽然堆排序平均时间最优，但实际中往往比 Sedgewick 增量序列的希尔排序**更慢**——缓存局部性差（元素到处跳）。

### 3.3 稳定性与原地性

- **Stable?** ❌ 不稳定。堆操作（下沉、交换）会打乱相等元素的相对顺序。
- **In-place?** ✅ 算法二是原地，额外空间 $O(1)$。算法一不是（空间翻倍）。

## 4. 归并排序（Mergesort）

### 4.1 核心操作：合并两个有序列表

```
A: 1  13  24  26          B: 2  15  27  38
   ↑Aptr                     ↑Bptr

C: 1  2  ...              ← 每次比较 A[Aptr] 和 B[Bptr]，取小的放入 C
   ↑Cptr
```

合并两个总长为 $N$ 的有序列表时间为 $O(N)$。

### 4.2 递归实现

```c
void MSort(ElementType A[], ElementType TmpArray[],
                  int Left, int Right) {
    int Center;
    if (Left < Right) {
        Center = (Left + Right) / 2;
        MSort(A, TmpArray, Left, Center);         /* T(N/2) */
        MSort(A, TmpArray, Center + 1, Right);    /* T(N/2) */
        Merge(A, TmpArray, Left, Center + 1, Right); /* O(N) */
    }
}
```

```c
void Merge(ElementType A[], ElementType TmpArray[],
                  int Lpos, int Rpos, int RightEnd) {
    int LeftEnd = Rpos - 1;
    int TmpPos = Lpos;
    int NumElements = RightEnd - Lpos + 1;

    while (Lpos <= LeftEnd && Rpos <= RightEnd)
        if (A[Lpos] <= A[Rpos])
            TmpArray[TmpPos++] = A[Lpos++];
        else
            TmpArray[TmpPos++] = A[Rpos++];
    while (Lpos <= LeftEnd)   /* 复制左半剩余 */
        TmpArray[TmpPos++] = A[Lpos++];
    while (Rpos <= RightEnd)  /* 复制右半剩余 */
        TmpArray[TmpPos++] = A[Rpos++];
    for (int i = 0; i < NumElements; i++, RightEnd--)
        A[RightEnd] = TmpArray[RightEnd];  /* 拷回原数组 */
}
```

### 4.3 复杂度分析

$$
\begin{align*}
T(1) &= 1\\
T(N) &= 2T(\frac{N}{2}) + O(N)\\
&= 2^k T(\frac{N}{2^k}) + k \cdot O(N)\\
&= N \cdot T(1) + \log N \cdot O(N)\\
&= O(N + N\log N) = O(N\log N)
\end{align*}
$$

### 4.4 稳定性与原地性

- **Stable?** ✅ 稳定。Merge 时 `A[Lpos] <= A[Rpos]` 取左边，相等元素保持原始顺序。
- **In-place?** ❌ 不是原地。需要 $O(N)$ 的临时数组 `TmpArray[]`。虽然存在原地归并的算法，但非常复杂且常数大，实际不用。

!!! tip "归并排序的定位"
    归并排序需要线性额外内存且拷贝开销大，**很少用于内部排序**。但它对于**外部排序**（数据在磁盘上）非常有用。

## 5. 快速排序

- 已知的实际运行最快的排序算法
- 选择一个基准元素（枢轴 pivot），将数组分成两个子数组，左边的元素都小于等于基准元素，右边的元素都大于等于基准元素，然后对两个子数组进行快排、合并
- 选取 pivot
    - 错误方法：pivot = arr[0]（对于排好序的数组仍会消耗 $O(N^2)$ 的时间）
    - 安全方法：pivot = random element in arr
        - 但随机数生成也有开销
    - 三数中值分割法：pivot = (left + center + right) / 3
- 小数组
    - 对于小的 $N$（$N\leq 20$），快速排序慢于插入排序
    - 可以在递归到 $N$ 较小的情况下改为插入排序
- 最坏复杂度 $O(N^2)$
- 最优复杂度 $O(N\log N)$
- 平均复杂度 $O(N\log N)$
- **Stable?** ❌ 不稳定。partition 过程中远距离交换打乱相对顺序。
- **In-place?** ✅ 原地（递归栈空间另算，$O(\log N)$）。

## 6. 桶排序

- 如果输入数据都小于 $M$，则可以用一个大小为 $M$ 的数组来记录某个值出现了多少次，这个数组称为桶（bucket）
- 桶初始化为 0，遍历输入数据，将每个数据对应的桶加 1
- 最后遍历桶中的所有元素，对于 bucket[x] = y，将 x 输出 y 次
- 时间复杂度 $O(N+M)$

---

## 7. 基数排序

- 从低位（LSD，Least Significant Digit）到高位（MSB），对每一位进行进行排序
- 例如对 64, 8, 216, 512, 27, 729, 0, 1, 343, 125 进行排序
    - 第一轮，按个位数排序
        - 0, 1, 512, 343, 64, 125, 216, 27, 8, 729
    - 第二轮，按十位数排序
        - (0, 1, 8), (512, 216), (125, 27, 729), 343, 64
    - 第三轮，按百位数排序
        - (0, 1, 8, 27, 64), 125, 216, 343, 512, 729
    - 完成排序
- 时间复杂度 $O(P(N+B))$，其中 $P$ 为轮数，$N$ 为元素个数，$B$ 为桶个数
- **Stable?** ✅ 稳定（前提是每一位的排序算法稳定）。
- **In-place?** ❌ 不是原地，需要桶数组。

## 8. 各算法 Stable 与 In-place 总结

| 算法 | Stable | In-place | 备注 |
|------|--------|----------|------|
| Insertion Sort | ✅ | ✅ | 严格大于才移动，不破坏相等顺序 |
| Shell Sort | ❌ | ✅ | 远距离跳跃打乱相对顺序 |
| Heapsort | ❌ | ✅（算法二） | 堆操作打乱相对顺序 |
| Mergesort | ✅ | ❌ | `<=` 取左边保证稳定，但需要 $O(N)$ 辅助空间 |
| Quick Sort | ❌ | ✅ | partition 的远距离交换打乱顺序 |
| 基数排序 | ✅ | ❌ | 取决于每位的排序是否稳定，需要桶 |
