## 1. 基础概念

- 有限的边集 $E = E(G)$ 和点集 $V = V(G)$
- 无向图（undirected graph）：边没有方向
- 有向图（directed graph）：边有方向
- 不考虑自环（self loop）和多重边（multigraph）
- 完全图（complete graph）：任意两个点之间都有边
- $v_i$ 和 $v_j$ 之间有一条无向边，则称：
    - $v_i$ 和 $v_j$ 相邻（adjacent）
    - $(v_i, v_j)$ is incident on $v_i$ and $v_j$
- 有一条从 $v_i$ 到 $v_j$ 的有向边，则称：
    - $v_i$ is adjacent **to** $v_j$
    - $v_j$ is adjacent **from** $v_i$
    - $\langle v_i, v_j \rangle$ is incident on $v_i$ and $v_j$
- 子图（subgraph）：边集和点集都是原图的子集
- 路径、路径长度都不必多说
    - 简单路径（simple path）：路径上的点不重复
- 回路（cycle）：路径的起点和终点相同
- 对于无向图：
    - 如果存在一条从 $v_i$ 到 $v_j$ 的路径，则称 $v_i$ 和 $v_j$ 连通（connected）
    - 如果任意一对点都连通，则称这个无向图是连通的（connected graph）
    - 无向图的连通组件（connected component of an undirected G）：无向图的极大连通子图
        1. 子图 (Subgraph)：它是原图 $G$ 的一部分，由原图中的部分顶点（Vertices）和部分边（Edges）组成。
        2. 连通 (Connected)：在这个子图内部， **任意两个顶点之间都至少存在一条路径可以互相到达** 。也就是说，这个子图内部是四通八达的，没有任何节点是被完全孤立在这个子图之外的。
        3. 极大 (Maximal)："极大"的意思是，你不能再从原图中拉任何一个其他顶点加入到这个子图中了。如果你强行拉一个外面的顶点进来，这个子图就不再"连通"了（因为那个外面的顶点和现在的子图没有任何边相连）。它已经"尽可能大"地把所有能连通的节点都圈在一起了。
        - 也就是说，  ==如果图G本身是连通的图的话，那么这幅图的 connected component 只有一个，就是它自己== 。
- 树是联通的无环图
- DAG（directed acyclic graph）：有向无环图
- 对于有向图：
    - 如果存在一条从 $v_i$ 到 $v_j$ 的路径，则称 $v_i$ 可达（reachable）$v_j$
    - 如果任意一对点都可达，则称这个有向图是强连通的（strongly connected）
    - 强连通分量：有向图的极大强连通子图
    - 如果有向图不是强连通的，但它的基础图（边去掉方向得到的无向图）是连通的，则称这个有向图是弱连通的（weakly connected）
- 度数：
    - 对于无向图，节点 $v$ 的度数 $\mathrm{degree}(v)$ 是与 $v$ 相邻的节点数
    - 对于有向图，节点 $v$ 的入度数 $\mathrm{indegree}(v)$ 是以 $v$ 为终点的边数，出度数 $\mathrm{outdegree}(v)$ 是以 $v$ 为起点的边数
    - 边数等于所有节点的度数之和除以 2

### 1.1 图的表示

- 邻接矩阵：
    - 如果存在一条从 $v_i$ 到 $v_j$ 的边，则 $a_{ij} = 1$，否则 $a_{ij} = 0$
        - 如果说这个图是一个无向图，设这个邻接矩阵为 A，则 $A = A^T$，且对角线上的值全为 0
    - 对于非稠密图来说非常浪费空间
    - **空间优化：** 无向图的邻接矩阵是对称的，可以只存下三角部分，压缩为 1D 数组 `adj_mat[n(n+1)/2] = {a11, a21, a22, ..., an1, ..., ann}`，其中 $a_{ij}$ 的索引为 `i*(i-1)/2 + j`
- 邻接表：
    - 对于每个节点，存储一个链表来记录所有和它相邻的节点
    - 对于无向图会将每个边存储两次
    - 对于有向图无法遍历入度，需要额外存储一个 **逆邻接表** （inverse adjacency lists）
- **邻接多重表（Adjacency Multilists）：**
    - 针对邻接表中每条无向边被存两次的问题，将两个节点合并为一个
    - 每个边节点包含：`mark`（是否被访问过）、`next_v1`（指向与 $v_1$ 相关的下一条边）、`next_v2`（指向与 $v_2$ 相关的下一条边）、`v1`、`v2`
    - 优点：访问某条边后可以方便地 `mark`，并快速找到下一条关联边，适合需要标记边的场景
- 带权图：
    - 邻接矩阵：$a_{ij}$ 存储边 $(v_i, v_j)$ 的权重
    - 邻接表/多重表：节点中增加 weight 字段

## 2. 拓扑排序（Topological Sort）

- AOV 网络：顶点表示活动，边表示活动之间的先后关系
    - 可以实现的 AOV 网络一定是 DAG（有向无环图）
- 如果存在一条从 $i$ 到 $j$ 的路径，则称 $i$ 是 $j$ 的前驱（predecessor），$j$ 是 $i$ 的后继（successor）
- 如果存在一条边 $\langle i, j \rangle$，则称 $i$ 是 $j$ 的直接（immediate）前驱，$j$ 是 $i$ 的直接后继
- 拓扑排序（topological order）是一个图的顶点集的线性序列，满足：
    - 对于任意一对点 $i, j$，如果 $i$ 是 $j$ 的前驱，则序列中 $i$ 在 $j$ 的前面

```c
void topsort(Graph G) {
    Queue Q = CreateQueue();
    int cnt = 0; Vertex V, W;
    for (each vertex V)
        if (indegree[V] == 0) Enqueue(Q, V);
    while (!IsEmpty(Q)) {
        V = Dequeue(Q);
        TopNum[V] = ++cnt;
        for (each W adjacent to V)
            if (--indegree[W] == 0) Enqueue(Q, W);
    }
    if (cnt != NumVertex) Error("Graph has a cycle");
    free(Q);
}
```

## 3. 最短路算法

**单源最短路（Single-Source Shortest Path）：** 给定一个带权有向图 $G = (V, E)$ 和源点 $s$，求从 $s$ 到图中每个顶点的最短路径（加权路径长度 $\sum_{e \in P} c(e)$）。

### 3.1 无权图的最短路

**朴素实现（O(|V|²)）：** 按距离从 0 开始，每轮遍历所有顶点找 `Dist == CurrDist` 的：

```c
void Unweighted(Table T) {
    int CurrDist;
    Vertex V, W;
    for (CurrDist = 0; CurrDist < NumVertex; CurrDist++) {
        for (each vertex V)
            if (!T[V].Known && T[V].Dist == CurrDist) {
                T[V].Known = true;
                for (each W adjacent from V)
                    if (T[W].Dist == Infinity) {
                        T[W].Dist = CurrDist + 1;
                        T[W].Path = V;
                    }
            }
    }
}
```

关键观察：如果 `V` 未知但 `Dist < Infinity`，则 `Dist` 一定是 `CurrDist` 或 `CurrDist+1`。基于这个观察，可以用队列优化：

```c
void Unweighted(Table T) {
    Queue Q = CreateQueue();
    Enqueue(S, Q);
    while (!IsEmpty(Q)) {
        V = Dequeue(Q);
        T[V].Known = true;
        for (each W adjacent to V)
            if (T[W].Dist == Infinity) {
                T[W].Dist = T[V].Dist + 1;
                T[W].Path = V;
                Enqueue(W, Q);
            }
    }
    free(Q);
}
```

- `T[i].Dist`：从 $s$ 到 $v_i$ 的距离（初始化为 $\infty$，$s$ 为 0）
- `T[i].Known`：是否已处理
- `T[i].Path`：记录路径前驱
- 时间复杂度：$O(|V| + |E|)$

### 3.2 Dijkstra 算法（带权最短路）

这份笔记中关于Dijkstra算法讲解的不是很详细，可以参考我在[CS61B的Lecture23中记录的关于Dijkstra算法的笔记](../../../CS61B/Lecture%2023%20Shortest%20Paths/Shortest%20Paths.md)，这一份笔记比较详细。

设 $S = \{s$ 和已找到最短路径的顶点$\}$。对任意 $u \notin S$，定义 `distance[u]` 为只经过 $S$ 中顶点从 $s$ 到 $u$ 的最短路径长度。

**贪心策略：** 每次选 `distance` 最小的 $u \notin S$ 加入 $S$，然后更新其邻居。

**正确性证明：** 假设最短路径经过某个不在 $S$ 中的顶点 $w$，那么该路径上必然有第一个不在 $S$ 中的顶点，其距离一定比 $u$ 更小——与 $u$ 是最小矛盾。

```c
void Dijkstra(Table T) {
    for (;;) {
        V = smallest unknown distance vertex;
        if (V == NotAVertex) break;
        T[V].Known = true;
        for (each W adjacent to V)
            if (!T[W].Known)
                if (T[V].Dist + Cvw < T[W].Dist) {
                    T[W].Dist = T[V].Dist + Cvw;
                    T[W].Path = V;
                }
    }
}
```

**两种实现方式：**

| 实现 | 找最小顶点 | Decrease | 总复杂度 | 适用场景 |
|------|-----------|----------|---------|---------|
| 1 | 线性扫描 $O(\lvert V\rvert)$ | $O(1)$ | $O(\lvert V\rvert^2 + \lvert E\rvert)$ | 稠密图 |
| 2 | 优先队列 $O(\log \lvert V\rvert)$ | DecreaseKey $O(\log \lvert V\rvert)$ 或直接插入 | $O(\lvert E\rvert \log \lvert V\rvert)$ | 稀疏图 |

### 3.2.1 时间复杂度推导

**方法 1（线性扫描）：** 共 $\lvert V\rvert$ 轮，每轮做两件事：

- 扫描所有未知顶点找最小距离 → 每轮 $O(\lvert V\rvert)$，$\lvert V\rvert$ 轮合计 $O(\lvert V\rvert^2)$
- 更新选中顶点的所有邻居 → 每条边恰好被松弛一次，$\lvert V\rvert$ 轮合计 $O(\lvert E\rvert)$
- 总复杂度：$O(\lvert V\rvert^2 + \lvert E\rvert) = O(\lvert V\rvert^2)$

**方法 2（优先队列）：**

- ExtractMin：每个顶点恰好出队一次，每次 $O(\log \lvert V\rvert)$，共 $O(\lvert V\rvert \log \lvert V\rvert)$
- DecreaseKey / Insert：每条边最多触发一次（松弛成功时），每次 $O(\log \lvert V\rvert)$，共 $O(\lvert E\rvert \log \lvert V\rvert)$
- 总复杂度：$O((\lvert V\rvert + \lvert E\rvert) \log \lvert V\rvert) = O(\lvert E\rvert \log \lvert V\rvert)$（连通图中 $\lvert E\rvert \geq \lvert V\rvert - 1$）

- **方法 2 的 DecreaseKey 实现：** 用堆维护未访问顶点，更新时调用 DecreaseKey（需维护顶点在堆中的位置）
- **方法 2 的插入实现：** 不修改堆内元素，直接将更新后的 `{W, newDist}` 插入堆，出队时跳过已访问的顶点。需要 $O(\lvert E\rvert)$ 空间。

!!! tip
    其他优化实现：Pairing heap（Ch.12）和 Fibonacci heap（Ch.11）可以进一步降低均摊复杂度。

注意：Dijkstra 不能处理负权边！

### 3.3 带负权边的图

遇到带负权边的图，Dijkstra算法就失效了，我们需要用到Bellman-Ford算法或者SPFA算法来解决，由于FDS这门课对于这一块的讲解实在是太不深入了，根本不讲这个算法为什么能够这样实现，也不讲解为什么Bellman-Ford算法执行了V次之后还能更新就说明有负环，SPFA也一样，所以我在[这里做了比较详细的笔记](../../补充知识/Bellman-Ford算法与SPFA算法.md)，希望能加深你们的理解。

**不能简单给每条边加常数** （因为边数不同的路径受影响不同，最短路径可能改变）。

!!! example "反例"
    ```
    1 → 2  边权 1
    1 → 3  边权 2
    3 → 2  边权 -2
    ```
    给每条边加 2 后：`1→2` 路径边数 1 总权变为 3，`1→3→2` 路径边数 2 总权变为 2。原本最短路径改变，加常数后最短路径不再正确。

#### 3.3.1 Bellman-Ford算法
[参考这个视频](https://www.bilibili.com/video/BV1jg411q7F8?spm_id_from=333.788.videopod.sections&vd_source=d6e21125b4ff2bde83430369b29c9418)

- 初始化源点距离为 0，其余为 ∞
- 重复 V−1V-1V−1 次：
    - 遍历所有边 $(u,v)$
    - 若 `dist[v] > dist[u] + w`，则更新
- 再遍历所有边一次：
    - 若还能更新 → 存在负环

##### 时间复杂度

- 每轮扫描所有边：$O(E)$
- 共 $V−1$ 轮

因此时间复杂度为 $O(VE)$
#### 3.3.2 SPFA算法（对Bellman-Frod进行了优化）
[参考这个视频](https://www.bilibili.com/video/BV1Xe4y1W7tB?spm_id_from=333.788.videopod.sections&vd_source=d6e21125b4ff2bde83430369b29c9418)


- 初始化源点 dist=0，入队
- 当队列不空：
    - 取出队首 u
    - 遍历 u 的所有邻边
    - 若能松弛 v：
        - 更新 $dist[v]$
        - 若 v 不在队列中，则入队
- 若某点入队次数 ≥ V → 可判负环

```c
void WeightedNegative(Table T) {
    Queue Q = CreateQueue();
    Enqueue(S, Q);
    while (!IsEmpty(Q)) {
        V = Dequeue(Q);
        for (each W adjacent to V)
            if (T[V].Dist + Cvw < T[W].Dist) {
                T[W].Dist = T[V].Dist + Cvw;
                T[W].Path = V;
                if (W is not already in Q)
                    Enqueue(W, Q);
            }
    }
}
```

- 与 BFS 的区别：同一顶点可能 **多次入队** （因为距离可能被多次更新）
- 每个顶点最多出队 $\lvert V\rvert-1$ 次
- 时间复杂度：$O(\lvert V\rvert \times \lvert E\rvert)$
- 负环会导致无限循环 （没有最短路径）

!!! explanation "分析时间复杂度"
    我之前一直不明白这个时间复杂度为什么是$O(|V| \times |E|)$，因为我认为，所有顶点最多都有可能入队V-1次，一共有V个顶点，因此总共的入队次数最多位V(V-1)次，每个顶点最多都连了E条边，因此我觉得时间复杂度是$O(|V|^2 \times|E|)$。大错特错了。
    
    每个顶点最多入队V-1次，一共V个顶点，所以总共的入队次数最多为V(V-1)次，这个是正确的，下面分成两种情况讨论：
    
    1. 有向图：若有向图中所有顶点恰好分别入队一次的话，总共就是正好把所有边都检查了一遍也就是E次，因为所有顶点都有可能入队V-1次，所以此时的最坏情况是 $E(V-1)$。
    2. 无向图：若无向图中所有的顶点都正好入队一次的话，那么每条边都会被检查两次，也就是总共检查了2E次，再乘以V-1就是 $2E(V-1)$.


### 3.4 有向无环图（DAG）

如果图是 DAG，可以按 **拓扑排序** 的顺序处理顶点：

```c
// 拓扑排序后依次松弛
for (each vertex v in topological order) {
    for (each W adjacent to v)
        if (dist[v] + Cvw < dist[W]) {
            dist[W] = dist[v] + Cvw;
            path[W] = v;
        }
}
```

- 时间复杂度：$O(\lvert V\rvert + \lvert E\rvert)$， **不需要优先队列**
- 原理：拓扑序保证处理 $v$ 时，所有可能的入边起点都已经处理完

### 3.5 AOE 网络（Activity On Edge）

[这个视频](https://www.bilibili.com/video/BV1fN411s7kf/?spm_id_from=333.337.search-card.all.click&vd_source=d6e21125b4ff2bde83430369b29c9418)讲的很清楚，用的例子跟浙大的PPT是一样的，感觉浙大的PPT上讲错了。

AOE 网络用于 **项目调度** ：边表示活动（持续时间 = 边权），顶点表示事件的完成。该方法也称为 **CPM**（Critical Path Method，关键路径法）。

- **EC[v]** （最早完成时间）：从起点正向计算，$EC[w] = \max_{\langle v,w\rangle \in E} \{ EC[v] + C_{v,w} \}$
- **LC[v]** （最晚完成时间）：从终点反向计算，$LC[v] = \min_{\langle v,w\rangle \in E} \{ LC[w] - C_{v,w} \}$
- 边 $\langle v,w \rangle$ 的 **松弛时间（slack time）** ：$LC[w] - EC[v] - C_{v,w}$
- **关键路径（critical path）** ：由松弛时间全为 0 的边构成的路径



!!! tip
    计算时如果存在多个前驱/后继，EC 取 max，LC 取 min。注意先拓扑排序确定计算顺序。

### 3.6 全源最短路（All-Pairs Shortest Path）

求任意两点之间的最短路径：

| 方法 | 思路 | 复杂度 | 适用 |
|------|------|--------|------|
| 单源重复 $\lvert V\rvert$ 次 | 对每个顶点跑一次单源最短路 | $O(\lvert V\rvert^3)$ | 稀疏图 |
| Floyd-Warshall | 动态规划 | $O(\lvert V\rvert^3)$ | 稠密图 |

!!! explanation "Princeton Algorithm: 最大流与最小割"
    ## 1. 最大流最小割定理与三个等价条件
    
    对于网络中的任意流 $f$，以下三个条件在数学上是等价的：

    - 条件 i：存在一个割，其容量等于当前流 $f$ 的值 (There exists a cut whose capacity equals the value of the flow $f$)。
    - 条件 ii：$f$ 是一个最大流。
    - 条件 iii：对于 $f$，不存在增广路径。
    
    


### 4.1 最大流问题

给定一个有向图，每条边有一个容量。有两个特殊顶点：

- **源点** (source）$s$：流量的起点
- **汇点** (sink）$t$：流量的终点

![](附件/Pasted%20image%2020260518170029.png)

**目标：** 求从 $s$ 到 $t$ 的最大流量。

**关键约束：** 对于任意中间顶点 $v \notin \{s, t\}$， **流入总量 = 流出总量** （流量守恒）。

![](附件/Pasted%20image%2020260518170224.png)

### 4.2 三个重要概念

**（1）流网络 $G$（Flow Network）：** 原始图，每条边标有容量。

**（2）流图 $G_f$（Flow Graph）：** 记录当前每条边上已经有多少流量。

**（3）残量图 $G_r$（Residual Graph）：** 记录每条边还能增加多少流量，以及可以"撤销"多少已有流量。


### 4.3 Naive Algorithm（不能保证最优性）

[这个教程讲的特别好](![](附件/Pasted%20image%2020260518170224.png))，Up主描述了Maximum Flow Problem并且阐述了Naive算法的实现过程以及它为什么不一定能找到最大流。

**步骤：**

1. 在残量图 $G_r$ 中找任意一条从 $s$ 到 $t$ 的路径（不能有回路）
2. 取这条路径上最小容量的边作为这条路径的流量
3. 把这个流量加到流图 $G_f$ 中，更新残量图 $G_r$（去掉流量为 0 的边）
4. 重复直到 $G_r$ 中不存在从 $s$ 到 $t$ 的路径

!!! danger "为什么朴素贪心会失败？"
    选择路径的顺序不同可能会产生不同的结果，以下面这幅图为例：
    ![](附件/Pasted%20image%2020260518171341.png)
    
    首先，通过刚才那个视频（或者自己推一下）可以知道这幅图的最大流是5。下面我给出一种路径：
    
    我选择`s->a->d->t`作为第一条路径，这条路径最大能装3个单位水。执行完这个操作之后，我们会发现没有路径可选了，因为Residual Graph中`d->t`已经为0了。而b又没法通过c到达t。
    
    因此，选择`s->a->d->t``作为第一条路径的话，得出的答案就是3，并非最大流。所以Naive Algorithm不一定能得出最大流。


### 4.4 正确算法：Ford-Fulkerson Algorithm

[这个视频讲解了Ford-Fulkerson算法](https://www.bilibili.com/video/BV1Pv41157xh?spm_id_from=333.788.player.switch&vd_source=d6e21125b4ff2bde83430369b29c9418&trackid=web_related_0.router-related-2479604-9kkcc.1779094591900.154)，也是上一个视频的Up主讲的，他分了好几集，循序渐进。

**Naive算法的致命缺陷：** 走不了回头路，没法撤回不好的选择。

Ford-Fulkerson算法和Naive算法非常相似，两者都用到了Residual Graph。唯一的区别是，Navies算法中的Residual Graph仅仅只是改变了每条边的权重，而Ford-Fulkersion算法在改变了权重之后还会加一条带权重的反向边，意思是: ==如果之前选择的路径不好，我可以倒着流回去== 。

**核心改进：** 对于流图 $G_f$ 中每条边 $(v, w)$ 已有流量 $f_{v,w}$，在残量图 $G_r$ 中 **添加一条反向边** $(w, v)$，容量也为 $f_{v,w}$。

**这意味着：** 算法可以"撤销"之前做的决定——反向边的存在相当于允许把之前流过 $v \to w$ 的流量退回去，改走别的路。

**完整算法步骤（Ford-Fulkerson 思想）：**

1. 在残量图 $G_r$ 中找一条从 $s$ 到 $t$ 的增广路径
2. 取路径上最小残量作为增广流量
3. 更新流图 $G_f$（正向边加流量，反向边记录可退流量）
4. 更新残量图 $G_r$（正向边减少残量，反向边增加残量）
5. 重复直到 $G_r$ 中不存在从 $s$ 到 $t$ 的路径

!!! proof "为什么加反向边就能保证正确？"
    反向边的存在让算法能够"反悔"——如果之前选了一条不好的路径，可以通过反向边把流量"推回去"，重新分配。

    可以证明：如果边的容量都是有理数，这个算法 **一定终止于最大流** 。

### 4.5 Ford-Fulkerson算法的复杂度分析（容量为整数时）

Ford-Fulkerson 算法的时间复杂度为 $O(f \cdot |E|)$，其中 $f$ 代表最大流， $|E|$ 代表边的数量。

**总时间复杂度 = (每次寻找增广路径的时间) $\times$ (寻找增广路径的最大总次数)**

#### 第一步：单次迭代（寻找路径 + 更新网络）的时间

在 Ford-Fulkerson 算法的一轮迭代中，我们需要做两件事：

1. **在残存网络中找一条从源点 $s$ 到汇点 $t$ 的增广路径。**
    - 这通常通过图的遍历算法来实现，比如深度优先搜索（DFS）或广度优先搜索（BFS）。
    - 对于一个有 $|V|$ 个顶点和 $|E|$ 条边的图，DFS 或 BFS 的时间复杂度是 $O(|V| + |E|)$。
    - 由于在一个连通的流网络中，边数通常大于等于顶点数减一（即 $|E| \ge |V| - 1$），所以遍历图的时间可以简化为 $O(|E|)$。
2. **沿着这条路径更新流量和残存网络。**
    - 一条简单路径最多包含 $|V|$ 个顶点，也就是最多 $|V| - 1$ 条边。因此，更新正向边和反向边的权重只需遍历这条路径即可，时间复杂度为 $O(|V|)$，它显然被包含在 $O(|E|)$ 之中。
    
**结论：** 单次迭代（找一次增广路径并更新）的时间复杂度是 **$O(|E|)$**。

#### 第二步：最大迭代次数的上限

这里就是最大流的值 $f$（通常记作 $|f^*|$）发挥作用的地方。

1. 算法的初始流量为 $0$。
2. 因为所有边的容量都是整数，所以我们每次找到的增广路径的“瓶颈容量”（路径上最小的剩余容量）**至少是 $1$**。
    
3. 这意味着，每一次成功的迭代，总流量至少会增加 $1$ 个单位。
    
4. 既然最大流的总量是 $f$，那么每次至少加 $1$，最多需要加多少次才能达到 $f$ 呢？显然， **最多需要迭代 $f$ 次** 。
    

**结论：** 算法最多执行 **$f$** 次迭代。

### 第三步：得出总复杂度

将第一步和第二步的结论相乘：

$$ \text{总时间} \le \text{最大迭代次数} \times \text{单次迭代代价} $$

$$ \text{总时间} = O(f) \times O(|E|) = O(f \cdot |E|) $$

### 4.6 改进时间复杂度的方法
由于Ford-Fulkerson算法比较死板，像下面这幅图，f=4000000，完全没必要在a和b之间穿梭那么多次，因此下面有两种改进的算法：

![](附件/Pasted%20image%2020260518184532.png)
#### 第一种改进的方法：
找路径的方法：修改Dijkstra

![](附件/Pasted%20image%2020260518182401.png)

- **具体做法：** 不再用普通的 DFS 瞎找，而是使用修改版的 Dijkstra 算法。原本 Dijkstra 是找“距离之和最小”的路径，现在把它改成找“路径上最小容量（瓶颈）最大”的路径。
- **如何化解危机：** 在图示的极端情况中，算法第一眼就会看中 $s \to a \to t$（瓶颈为 1,000,000）和 $s \to b \to t$（瓶颈为 1,000,000），直接两步到位，根本不会去碰中间那条容量为 1 的细管子。
- **时间复杂度拆解：**
    - **寻找一次路径的时间 ($T_{find\_a\_path}$)：** 跑一次基于堆优化的修改版 Dijkstra，复杂度是 $O(|E| \log |V|)$。
    - **最大迭代次数 ($T_{augmentation}$)：** 数学上可以证明，如果每次都挑最大容量路径，最多只需要迭代 $O(|E| \log cap_{max})$ 次。
    - **总复杂度：** $O(|E|^2 \log cap_{max} \log |V|)$。幻灯片最后指出，如果最大容量的对数 $\log(cap_{max})$ 相对较小，可以视作常数，那么复杂度就可以简化为 **$O(|E|^2 \log |V|)$**。
#### 第二种改进的方法：
找路径的方法：BFS（无权最短路径）
![](附件/Pasted%20image%2020260518182504.png)

- **具体做法：** 使用无权图最短路径算法，也就是我们常说的 **BFS（广度优先搜索）** 来寻找路径。
- **如何化解危机：** 在那张杀手图中，如果用 BFS，第一层会扫到 $a$ 和 $b$，第二层就会扫到 $t$。所以算法会优先找到仅包含 2 条边的路径（$s \to a \to t$ 或 $s \to b \to t$），而中间那条包含 3 条边的交叉路径（$s \to a \to b \to t$）因为太长了，在 BFS 时优先级靠后，从而成功避开了陷阱。
- **时间复杂度拆解：**
    - **寻找一次路径的时间 ($T_{find\_a\_path}$)：** 跑一次 BFS 只需要 **$O(|E|)$**。
    - **最大迭代次数 ($T_{augmentation}$)：** 这是一个非常著名的定理（由 Edmonds 和 Karp 证明）：只要每次用 BFS 找最短增广路，整个网络中的增广次数绝对不会超过 **$O(|V| \cdot |E|)$** 次。这个上限彻底摆脱了具体流量数值的控制！
    - **总复杂度：** 将两者相乘，得出总时间复杂度为 **$O(|V| \cdot |E|^2)$**。

#### 总结

| 增广路径选择策略   | 找路径方法       | 单次增广复杂度                                      | 总复杂度                                      |
| ---------- | ----------- | -------------------------------------------- | ----------------------------------------- |
| 任意路径       | BFS/DFS     | $O(\lvert E\rvert)$                          | $O(f \cdot \lvert E\rvert)$，$f$ 为最大流的数量   |
| 最大增广（流量最多） | 修改 Dijkstra | $O(\lvert E\rvert \log \mathrm{cap}_{\max})$ | $O(\lvert E\rvert^2 \log \lvert V\rvert)$ |
| 最少边数路径     | BFS（无权最短路）  | $O(\lvert E\rvert)$                          | $O(\lvert E\rvert^2 \lvert V\rvert)$      |

!!! tip "特殊情况优化"
    如果每个中间节点 $v \notin \{s, t\}$ 要么只有一条入边（容量 1），要么只有一条出边（容量 1），时间复杂度可降到 $O(\lvert E\rvert \lvert V\rvert^{1/2})$。

!!! question "我的疑问"
    我在听这个算法的时候一直有一个疑问：Ford-Fulkerson算法之所以在改变了Residual Graph之后还加上了反向权重边，不就是为了供我们反悔用的吗？如果要反悔的话，不应该是把原来的一整条路径都反悔了吗？为什么只反悔一条边也能成立？
    
    ### Gemini的解答：
    反向边并不是真正意义上的“把原路的水抽回来”，而是一种“流量置换”（或者叫“改道”）。 网络流算法并不“记忆”你之前走过的是哪一整条路径，它只关心每个节点当前的流量守恒。
    
    我们可以通过一个经典的“水管网”例子来完美解释这个过程。
    
    ### 经典的“狸猫换太子”模型
    
    假设我们有 4 个节点：源点 $S$，中间节点 $A$ 和 $B$，汇点 $T$。每条水管的容量都是 1：
    
    - $S \rightarrow A$ (容量 1)
    - $S \rightarrow B$ (容量 1)
    - $A \rightarrow B$ (容量 1)
    - $A \rightarrow T$ (容量 1)
    - $B \rightarrow T$ (容量 1)
    
    #### 第一次寻找路径（走弯路了）
    
    假设算法是个“近视眼”，它第一次找到的增广路径是：**$S \rightarrow A \rightarrow B \rightarrow T$**，流量为 1。

    现在真实的物理水流是这样的：

    1. 水从 $S$ 流到 $A$。
    2. 水从 $A$ 流到 $B$。
    3. 水从 $B$ 流到 $T$。
    
    此时，在**残存网络**中，为了允许反悔，算法建立了一条**反向边 $B \rightarrow A$**，容量为 1。

    #### 第二次寻找路径（利用反向边“反悔”）
    
    现在，算法在残存网络中继续找路径。它发现了一条新路径：$S \rightarrow B \rightarrow A \rightarrow T$ （注意，这里借用了反向边 $B \rightarrow A$）。
    
    **你的疑问就在这里：** 我们只利用了反向边 $B \rightarrow A$，相当于只撤销了 $A \rightarrow B$ 的流量。那原来 $S \rightarrow A$ 和 $B \rightarrow T$ 的水怎么办？它们不会断掉吗？
    
    我们来看看在物理层面上，各个节点到底发生了什么“交易”：
    
    1. **在节点 A 处：**
        - 原来 $A$ 从 $S$ 接收了 1 份水，把它交给了 $B$。
        - 现在，新的路径要求水从 $A$ 流向 $T$。于是 $A$ 顺水推舟：“既然现在有路通向 $T$ 了，那我原来从 $S$ 接来的水，**就不给 $B$ 了，直接给 $T$ 吧**。”
        - 👉 结果：形成了一条完整的真实路径 **$S \rightarrow A \rightarrow T$**。
    
    2. **在节点 B 处：**
        - 原来 $B$ 负责把从 $A$ 那里接收的 1 份水交给 $T$。
        - 现在，新的路径让 $B$ 直接从 $S$ 接收了 1 份水。于是 $B$ 告诉 $A$：“哥们，我现在直接从源头 $S$ 拿到水了，**我不需要你的水了**，我把我自己的水给 $T$ 就行。”
        - 👉 结果：形成了另一条完整的真实路径 **$S \rightarrow B \rightarrow T$**。
    
    #### 结论：为什么不需要撤销一整条边？

    当我们沿着反向边 $B \rightarrow A$ 增加流量时，实际上是发生了**两股水流的重新配对**：

    - $S \rightarrow A$ 原本和 $A \rightarrow B \rightarrow T$ 配对；
    - $S \rightarrow B$ 原本无路可走。
    - 通过反向边抵消掉 $A \rightarrow B$ 后，$S \rightarrow A$ 重新和 $A \rightarrow T$ 配对；$S \rightarrow B$ 重新和 $B \rightarrow T$ 配对。
    
    这就像是你原本左手拿苹果，右手拿香蕉。反悔并不需要你把水果都扔回果园重新摘，只需要让你的左手和右手**交换一下目标**即可。因为中间的 $A \rightarrow B$ 被抵消了，两端的多余流量自然会顺着新路径找到出口，完美保持了流量守恒！

## 5. 最小生成树（MST）

FDS这里讲的实在是太简略了，连证明都没有直接讲算法，这边建议参考我在[CS61B的lecture 24中的笔记](../../../CS61B/Lecture%2024%20Minimum%20Spanning%20Trees/Lecture%2024%20Minimum%20Spanning%20Trees.md)，CS61B中有很详细的证明过程。

### 5.1 定义

**生成树（Spanning Tree）：** 包含图 $G$ 的所有顶点和一部分边的 **树** （连通、无环）。

- 边数一定是 $\lvert V\rvert - 1$
- 一个图可能有多个不同的生成树
- MST 存在当且仅当图是 **连通的**

**最小生成树（MST）：** 总边权最小的生成树。

### 5.2 贪心策略

MST 算法都遵循以下贪心框架，每次做最优选择，满足三个约束：
1. 只能用图中的边
2. 恰好使用 $\lvert V\rvert - 1$ 条边
3. 不能产生环

### 5.3 Prim 算法（"长"出一棵树）

从一个任意起点开始，反复选择 **一端在 MST 内、一端在 MST 外** 的权重最小的边，加入 MST。

**与 Dijkstra 极其相似：** Dijkstra 考虑"距离源点有多近"，Prim 考虑"距离当前 MST 有多近"。

**复杂度：** $O(\lvert E\rvert \log \lvert V\rvert)$（优先队列实现）

### 5.3.1 时间复杂度推导

Prim 的结构与 Dijkstra 完全一致，区别仅在于距离的定义：

- ExtractMin：每个顶点恰好加入 MST 一次，每次 $O(\log \lvert V\rvert)$，共 $O(\lvert V\rvert \log \lvert V\rvert)$
- DecreaseKey：检查每条边是否比当前到 MST 的最短边更短，每条边最多触发一次，每次 $O(\log \lvert V\rvert)$，共 $O(\lvert E\rvert \log \lvert V\rvert)$
- 总复杂度：$O((\lvert V\rvert + \lvert E\rvert) \log \lvert V\rvert) = O(\lvert E\rvert \log \lvert V\rvert)$

### 5.4 Kruskal 算法（维护一片森林）

```c
void Kruskal(Graph G) {
    T = { };
    while (T contains less than |V|-1 edges && E is not empty) {
        choose a least cost edge (v, w) from E;  /* DeleteMin */
        delete (v, w) from E;
        if ((v, w) does not create a cycle in T)  /* Union/Find */
            add (v, w) to T;
        else
            discard (v, w);
    }
    if (T contains fewer than |V|-1 edges)
        Error("No spanning tree");
}
```

1. 按权重从小到大考虑每条边
2. 如果加入不会形成环，就加入 MST
3. 重复直到有 $\lvert V\rvert - 1$ 条边

**通过并查集判断环** ，复杂度：$O(\lvert E\rvert \log \lvert E\rvert)$

## 6. 深度优先搜索（DFS）

- 深度优先搜索（Depth-First Search，DFS）是一种用于遍历或搜索树或图的算法
- 本质上是`preorder traversal`在一般图上的推广
- 核心特点：
    - 用递归实现
    - **回溯：** 走到死胡同时退回上一步
    - 用 `visited[]` 标记已访问顶点，避免重复访问和死循环

### 6.1 DFS 的基本实现与边的分类

#### 算法框架

```c
/* 初始化 */
for (each v in V)
    visited[v] = false;

/* 对每个未访问顶点启动 DFS */
for (each v in V)
    if (!visited[v])
        Explore(v);
```

```c
/* 核心递归函数 */
int count = 0;  // DFS 序号计数器
int Num[MAXV];   // Num[v] = DFS 访问序号（第几个被访问）

void Explore(Vertex v) {
    count++;
    Num[v] = count;       // 分配 DFS 序号
    visited[v] = true;
    
    for (each neighbor u of v) {
        if (!visited[u]) {
            parent[u] = v;        // 记录 parent，用于区分边类型
            Explore(u);
        } else if (u != parent[v]) {
            // 找到一条反向边 (back edge)！
            // u 已经被访问且不是 v 的 parent → 是 v 的祖先
            // 在这里可以根据需要记录/处理这条反向边
            // 例如：printf("Back edge: (%d, %d)\n", v, u);
        }
    }
}
```

时间复杂度：$O(\lvert V\rvert + \lvert E\rvert)$（使用邻接表时）

#### DFS 编号 Num(v)

- `Num(v)` = DFS 过程中第几个访问到 $v$（从 1 到 $|V|$）
- Num 值越小，说明在 DFS 树中越靠"上"（越早被访问）

#### 边的分类（无向图）

DFS 将无向图的边分为两类：

| 类型        | 定义                                      | 示意图 |
| --------- | --------------------------------------- | --- |
| Tree Edge | DFS 过程中首次发现新顶点的边，构成 **DFS 树 / 深度优先生成树** | 实线  |
| Back Edge | 连接一个顶点和它在 DFS 树中的祖先（不包括 parent）的边       | 虚线  |


!!! warning "无向图的特殊性质"
    无向图的 DFS **只有** tree edge 和 back edge，**不存在** cross edge（横跨不同分支的边）。
    
    这是因为无向图中边是双向的，如果 $(u,v)$ 是一条边，DFS 访问 $u$ 时一定会顺着 $(u,v)$ 访问到 $v$（或发现 $v$ 已经被祖先访问过 → back edge）。

**检测反向边的方法：** 遍历边 $(v, u)$ 时，如果 `visited[u] == true` 且 `u ≠ parent[v]`，则 $(v, u)$ 是反向边。

> 这里追踪 `parent[v]` 很关键——如果只检查 `visited[u]`，会把 tree edge 的回退也误判为 back edge。

---

### 6.2 连通分量

对无向图求所有连通分量，只需对每个未访问的顶点启动一次 DFS：

```c
void ListComponents(Graph G) {
    for (each V in G)
        if (!Visited[V]) {
            DFS(V);
            printf("\n");  // 当前连通分量结束
        }
}
```

每调用一次 `DFS(V)`，就会遍历完 $V$ 所在的整个连通分量。打印换行后，所有属于该分量的顶点就输出在了一起。

---

### 6.3 双连通性：Tarjan 算法找割点

#### 问题引入

在无向连通图中，有些顶点特别"脆弱"——把它删掉之后，图就裂成了好几块。这种顶点叫**割点**。比如计算机网络中，割点就是"单点故障"——路由器坏了整个网络就断了。

Tarjan 算法用**一次 DFS**（$O(|V|+|E|)$）就能找出所有割点。要做到这一点，需要引入两个核心概念：**Num**（DFS 序号）和 **Low**（追溯值）。

#### 第一步：跑一次 DFS，得到 DFS 树 + Num 值

选一个起点开始 DFS，走过的是**树边**（实线），构成 **DFS 树**（也叫深度优先生成树）。沿途给每个顶点一个编号 `Num[v]` = 第几个被访问到。

DFS 过程中可能遇到一些"已经访问过但不是 parent"的邻居——这些边不在 DFS 树上，称为**反向边**（虚线），它**总是连接祖先和后代**。

!!! example "举个例子"
    假设图：A—B—C—D，外加 A—C、B—D。从 A 开始 DFS：
    
    ```
    走法: A → B → C → D
    Num:  1    2    3    4
    
    树边（实线）: (A,B), (B,C), (C,D)
    反向边（虚线）: A—C（C 连回祖先 A）、B—D（D 连回祖先 B）
    ```
    
    注意：D 访问完后回溯到 C，C 虽然没有未访问邻居了，但 C 有一条反向边连到 A（Num=1）——这意味着从 C **可以不经过 B** 直接跳回 A。

#### 第二步：定义 Low[v]——"能追溯到哪里"

**核心问题：** $v$ 及其后代，走若干步树边（向下）+ **至多一条**反向边（向上跳），能到达的**最小 Num** 是多少？

把这个最小值记为 `Low[v]`。它衡量的是"$v$ 这棵子树有多大的'自由度'绕过上方顶点"。

$$
\mathrm{Low}(u) = \min \begin{cases}
\mathrm{Num}(u) & \text{① 自己本身} \\[6pt]
\displaystyle\min_{(u, w) \text{ 是反向边}} \mathrm{Num}(w) & \text{② 直接通过一条反向边跳到某祖先} \\[6pt]
\displaystyle\min_{w \text{ 是 } u \text{ 的儿子}} \mathrm{Low}(w) & \text{③ 让儿子先往下走，儿子能到哪我就能到哪}
\end{cases}
$$

三种情况对应三种"自由度来源"：
- ① 我自己的位置（兜底，哪儿也跳不过去）
- ② 我直接有一条反向边连到某个祖先，Num 很小 → Low 就小
- ③ 我儿子本身就有反向边能跳到更高的地方，那我也能通过儿子绕上去

**在 DFS 递归中计算 Low：**

| 时机 | 操作 |
|------|------|
| 刚进入 $u$ | `Low[u] = Num[u]`（初始化为自己的 Num） |
| 遍历邻居 $w$，$(u,w)$ 是**树边** | 先 `DFS(w)`，回溯时 `Low[u] = min(Low[u], Low[w])` |
| 遍历邻居 $w$，$(u,w)$ 是**反向边** | `Low[u] = min(Low[u], Num[w])` |

!!! example "接上面的例子，算 Low"
    ```
    图: A—B—C—D，反向边 A—C、B—D
    从 A 开始 DFS → A(1)→B(2)→C(3)→D(4)
    
    回溯计算 Low:
    D: Low[D]=4，无反向边，无儿子 → Low[D]=4
    C: Low[C]=3，有反向边(C,A)→Low[C]=min(3,1)=1
       儿子 D → Low[C]=min(1, Low[D]=4)=1 → Low[C]=1
    B: Low[B]=2，有反向边(B,D)→Low[B]=min(2,4)=2
       儿子 C → Low[B]=min(2, Low[C]=1)=1 → Low[B]=1
    A: Low[A]=1，儿子 B → Low[A]=min(1, Low[B]=1)=1 → Low[A]=1
    ```
    
    总结：Low[C]=1 说明从 C 出发能跳到 Num=1（即 A），不需要经过 B。Low[B]=1 说明从 B 也能（通过儿子 C 的反向边）跳到 A。

#### 第三步：用 Low 和 Num 判定割点

**直觉：** 考虑顶点 $u$ 和它的一个儿子 $w$。$w$ 的子树的 Low 值反映了它"不经过 $u$ 能跳多远"。

- 如果 $\mathrm{Low}(w) < \mathrm{Num}(u)$：$w$ 的子树可以通过某条反向边跳到 $u$ **上方** → 删掉 $u$ 也无所谓，$w$ 的子树和剩余图仍然连通
- 如果 $\mathrm{Low}(w) \geq \mathrm{Num}(u)$：$w$ 的子树**跳不上去**，最多跳到 $u$ 自己 → $u$ 是必经之路 → $u$ 是割点

| 顶点类型 | 判定条件 | 直观理解 |
|----------|----------|----------|
| **根节点**（DFS 起点） | 有 **≥ 2 个儿子** | 根上面没有祖先，各子树之间只能通过根连通 |
| **非根节点 $u$** | 存在儿子 $w$ 使 $\mathrm{Low}(w) \geq \mathrm{Num}(u)$ | $w$ 子树无法绕过 $u$ 跳到更高处，$u$ 是"咽喉要道" |

!!! warning "根节点的特判"
    根节点没有祖先可以跳，所以 Low ≥ Num 永远成立——不能用 Low(w) ≥ Num(root) 来判定。正确做法是：如果根节点在 DFS 树中有 **≥ 2 个儿子**，那么去掉根后各儿子所在子树彼此不连通，根是割点。

#### 完整伪代码

```c
void FindArt(Vertex v, int dfs_id) {
    Num[v] = Low[v] = ++dfs_id;   // 分配 DFS 序号，Low 初始 = Num
    visited[v] = true;
    
    for (each neighbor u of v) {
        if (!visited[u]) {         // 树边
            child_count++;
            parent[u] = v;
            FindArt(u, dfs_id);
            Low[v] = min(Low[v], Low[u]);   // 儿子能到哪我就能到哪
            
            // 割点判定（非根节点）
            if (parent[v] != NULL && Low[u] >= Num[v])
                v 是割点;
        } else if (u != parent[v]) {   // 反向边
            Low[v] = min(Low[v], Num[u]);
        }
    }
    
    // 根节点特判
    if (parent[v] == NULL && child_count >= 2)
        v 是割点;
}
```

---

#### 基本概念总结

- **割点（Articulation Point / Cut Vertex）：** 去掉这个点（以及相连的边）后，图不再连通
- **双连通图（Biconnected Graph）：** 连通且没有任何割点
- **双连通分量（Biconnected Component）：** 极大的双连通子图
    - 图的**边集**被双连通分量**划分**（没有边可以同时属于两个不同的双连通分量）
    - 但割点本身可以属于多个双连通分量

### 6.3 欧拉回路与欧拉路径

#### 定义

- **欧拉回路（Euler Circuit）：** 包含图中**所有边**的简单环（起点 = 终点，每条边恰好走一次）
- **欧拉路径（Euler Path / Trail）：** 包含图中**所有边**的简单路径（起点 ≠ 终点，每条边恰好走一次）

#### 判定条件

**无向图：**

| | 条件 |
|---|---|
| **欧拉回路存在** | 图连通，且**所有顶点的度数为偶数** |
| **欧拉路径存在** | 图连通，且**恰好两个顶点的度数为奇数**（这两个就是路径的起点和终点） |

**有向图：**

| | 条件 |
|---|---|
| **欧拉回路存在** | 图弱连通，且**每个顶点的出度 = 入度** |
| **欧拉路径存在** | 图弱连通，且恰好**一个顶点出度比入度大 1**（起点）、**一个顶点入度比出度大 1**（终点），其余出度 = 入度 |

#### 算法

**Fleury 算法（$O(|E|^2)$）：** 从起点出发，每次优先选**不是桥**的边（除非别无选择），走过即删除。走到无边可走时停止。

**DFS 后序翻转算法（$O(|V| + |E|)$，推荐）：**

核心思想：DFS 遍历，走一条边就删一条边。当某个顶点无路可走时，把它加入答案序列。最后**翻转序列**即得欧拉回路。

```c
void dfs(int x) {
    for (int y = 1; y <= maxn; ++y) {
        if (G[x][y]) {
            G[x][y] = 0;   // 删除边
            G[y][x] = 0;   // 无向图双向删除
            dfs(y);
        }
    }
    ans[++ansi] = x;   // 无路可走时加入答案
    return;
}
```

遍历完成后将 `ans[]` 翻转（reverse），即得到欧拉回路/路径的顶点序列。

!!! tip "为什么后序翻转？"
    DFS 后序遍历保证了：只有当一个分支被"走完"（没有未访问的边了），当前顶点才被记录。翻转后，就变成了"先把回路的主体走完，再把分支嵌入到正确的位置"——这就是 Hierholzer 算法的本质。

**主流程：**

```c
// 检查度数
for (int i = 1; i <= maxn; ++i) {
    if (deg[i] % 2) {
        cnt++;
        if (!root) root = i;   // 第一个奇度顶点作为起点
    }
}
if (!root) {   // 全为偶数度，从任意非零度顶点出发
    for (int i = 1; i <= maxn; ++i) {
        if (deg[i]) { root = i; break; }
    }
}
if (cnt && cnt != 2) {   // 奇度顶点不是 0 或 2 个 → 无解
    printf("No Solution\n");
    return 0;
}
dfs(root);
// ans[1..ansi] 翻转后即为欧拉回路/路径的顶点序列
```

### 6.4 Hamilton 回路

- **Hamilton 回路（Hamilton Cycle）：** 在无向图中找到一条简单环，访问图中的**每个顶点恰好一次**
- 与欧拉回路的区别：

| | 欧拉回路 | Hamilton 回路 |
|---|---|---|
| 要求 | 每条**边**恰好走一次 | 每个**顶点**恰好访问一次 |
| 顶点重复 | 可以 | 不可以 |
| 边 | 全部用到 | 不一定全部用到 |
| 判定 | 简单（度数奇偶性） | **NP-难** |

- Hamilton 回路问题是 NP 难的，没有像欧拉回路那样简单的判定条件
- 可以用 DFS **回溯**暴力搜索所有可能的路径（最坏指数时间）
