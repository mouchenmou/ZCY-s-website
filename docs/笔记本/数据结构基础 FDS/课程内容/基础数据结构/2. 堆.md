# 堆（Heap）

!!! abstract
    二叉堆（优先队列）的基本概念、插入、删除和建堆操作。

---

## 1. 优先队列 ADT

### 1.1 什么是优先队列

普通队列是 FIFO（先进先出），**优先队列**是"优先级最高的先出"。

支持的操作：
- `Insert` — 插入一个元素
- `DeleteMin` — 删除并返回最小元素（如果是最大堆则是 `DeleteMax`）
- `FindMin` — 查看最小元素（不删除）

### 1.2 简单实现对比

| 实现方式    | Insert            | DeleteMin               | 性质             |
| ------- | ----------------- | ----------------------- | -------------- |
| 无序数组    | $\Theta(1)$（加到末尾） | $\Theta(N)$（扫描找最小 + 移位） | 简单粗暴           |
| 无序链表    | $\Theta(1)$（插到头部） | $\Theta(N)$（扫描找最小）      | 删除只需 $O(1)$ 移除 |
| 有序数组    | $O(N)$（找位置 + 移位）  | $\Theta(1)$（删第一个/最后一个）  | 插入慢            |
| 有序链表    | $O(N)$（找位置）       | $\Theta(1)$（删第一个/最后一个）  | 插入慢            |
| **二叉堆** | **$O(\log N)$**   | **$O(\log N)$**         | ✅ 最优           |

!!! tip "为什么不用 BST/AVL 树？"
    AVL 树 Insert 和 Delete 都是 $O(\log N)$，理论上也可以做优先队列。但：
    - 优先队列**永远只删除最小元素**，不需要 BST 的全部搜索能力
    - 堆没有指针开销，常数因子更小
    - 堆天然是完全二叉树 → 数组存储 → Cache 友好

## 2. 二叉堆的结构性质

### 2.1 完全二叉树

!!! conception "完全二叉树（Complete Binary Tree）"
    一棵 $n$ 个节点、高度为 $h$ 的二叉树，如果它的节点和**高度为 $h$ 的完美二叉树**中编号 $1 \sim n$ 的节点一一对应，则它是完全二叉树。

    简单理解：**从上到下、从左到右**逐个填满，中间不能有空位。

高度为 $h$ 的完全二叉树：

$$2^h \le N \le 2^{h+1}-1 \quad\Rightarrow\quad h = \lfloor \log N \rfloor$$

### 2.2 数组表示

完全二叉树可以用**数组**存储，不需要指针——这就是堆最大的优势。

!!! example "数组与树的对应"
    我们选择浪费掉`Key[0]`这个节点，从数组的下标为1开始存储元素，这样子表示父节点和左右儿子会一目了然。
    ![](附件/Pasted%20image%2020260602221410.png)

**下标 $i$ 的节点，它的——**

| 关系 | 下标 | 条件 |
|------|------|------|
| 父节点 | $\lfloor i/2 \rfloor$ | $i \neq 1$ |
| 左儿子 | $2i$ | $2i \le N$ |
| 右儿子 | $2i+1$ | $2i+1 \le N$ |

!!! warning "注意"
    数组下标从 **1** 开始，BT[0] 留作哨兵（sentinel）。

## 3. 堆序性质

!!! conception "最小堆（Min Heap）"
    每个节点的值**不大于**其所有子节点的值。根就是全堆最小的元素。
    
    反过来，**最大堆（Max Heap）**：每个节点**不小于**子节点，根是最大的。

```
  最大堆:             最小堆:
    9                  10
   / \                /  \
  6   3             20   83
 /                  /
5                 50
```

!!! tip "应用选择"
    - 需要每次取**最小** → 最小堆（如 Dijkstra）
    - 需要每次取**最大** → 最大堆（如 Heapsort）

## 4. 基本操作

### 4.1 插入（Insert）— 上滤（Percolate Up）

**思路：** 新元素先放到**最后**（维持完全二叉树结构），然后**一路向上**跟父节点比较，比父节点小就交换，直到找到合适位置。


```c
/* H->Elements[0] 是哨兵，值 ≤ 堆中所有元素 */
void Insert(ElementType X, PriorityQueue H) {
    int i;
    if (IsFull(H)) {
        Error("Priority queue is full");
        return;
    }
    for (i = ++H->Size; H->Elements[i / 2] > X; i /= 2)
        H->Elements[i] = H->Elements[i / 2];  /* 父节点下沉 */
    H->Elements[i] = X;                        /* 放入 X */
}
```

- 哨兵 `H->Elements[0]` 放置 ≤ 最小元素的值——保证循环一定会停下来。
- 时间复杂度：$O(\log N)$

### 4.2 删除最小（DeleteMin）— 下滤（Percolate Down）

**思路：** 
1. 取出根节点（最小元素）
2. 把**最后一个元素**拎到根的位置
3. 一路向下跟**较小的儿子**比较，比儿子大就交换


```c
ElementType DeleteMin(PriorityQueue H) {
    int i, Child;
    ElementType MinElement, LastElement;
    if (IsEmpty(H)) {
        Error("Priority queue is empty");
        return H->Elements[0];
    }
    MinElement = H->Elements[1];              /* 保存最小值 */
    LastElement = H->Elements[H->Size--];     /* 取最后一个元素 */
    for (i = 1; i * 2 <= H->Size; i = Child) {
        Child = i * 2;                        /* 左儿子 */
        if (Child != H->Size
            && H->Elements[Child + 1] < H->Elements[Child])
            Child++;                          /* 右儿子更小，选它 */
        if (LastElement > H->Elements[Child])
            H->Elements[i] = H->Elements[Child]; /* 儿子上浮 */
        else
            break;                            /* 找到合适位置 */
    }
    H->Elements[i] = LastElement;
    return MinElement;
}
```

- 时间复杂度：$O(\log N)$

### 4.3 其他操作

| 操作 | 方法 | 复杂度 |
|------|------|--------|
| DecreaseKey(P, Δ) | 降低位置 P 的值，然后**上滤** | $O(\log N)$ |
| IncreaseKey(P, Δ) | 增大位置 P 的值，然后**下滤** | $O(\log N)$ |
| Delete(P) | DecreaseKey(P, ∞) + DeleteMin | $O(\log N)$ |

## 5. 建堆（BuildHeap）— 重点

!!! question "为什么需要 BuildHeap？"
    一个一个 Insert 建堆需要 $O(N\log N)$。有没有更快的方法？有——**$O(N)$ 建堆**。

### 5.1 核心思路

给定一个无序数组，我们**直接把它当成一棵完全二叉树**（不满足堆序），然后从最后一个非叶节点开始，**逐个向下过滤（PercolateDown）**，最终整棵树满足堆序。

### 5.2 怎么找第一个非叶节点？

**最后一个非叶子节点 = 最后一个节点的父节点。**

最后一个节点的下标是 $N$，它的父节点下标是：

$$\text{第一个非叶节点} = \lfloor N / 2 \rfloor$$

!!! example "例子：$N=15$"
    最后一个节点编号 = 15，它的父节点 = $\lfloor 15/2 \rfloor = 7$。
    
    所以从**编号 7 开始**，依次对 7, 6, 5, 4, 3, 2, 1 做 PercolateDown。
    
    编号 8~15 都是叶子节点——叶子不需要下沉。

### 5.3 完整示例

数组：`{150, 80, 40, 30, 10, 70, 110, 100, 20, 90, 60, 50, 120, 140, 130}`

```
初始当作完全二叉树（乱序）：
               150
           /        \
         80          40
       /    \       /    \
     30     10    70     110
    /  \   /  \   /  \   /  \
  100 20 90  60 50 120 140 130

N=15，第一个非叶节点 = 7，依次处理 7,6,5,4,3,2,1。
```

**对 7 (值为 110) 做 PercolateDown：**
- 儿子：14 (130), 15 (无)。110 < 130 → 不动。

**对 6 (值为 70) 做 PercolateDown：**
- 儿子：12 (50), 13 (120)。50 更小，70 > 50 → 70 下沉到 12，50 上浮到 6。
- 50 到位置 12 后没有儿子，停止。

**对 5 (值为 10) 做 PercolateDown：**
- 儿子：10 (90), 11 (60)。60 更小，10 < 60 → 不动。（已经是小顶堆）

**对 4 (值为 30) 做 PercolateDown：**
- 儿子：8 (100), 9 (20)。20 更小，30 > 20 → 30 下沉到 9，20 上浮到 4。
- 30 在位置 9 无儿子，停止。

**对 3 (值为 40) 做 PercolateDown：**
- 儿子：6 (50), 7 (110)。50 更小，40 < 50 → 不动。

**对 2 (值为 80) 做 PercolateDown：**
- 儿子：4 (20), 5 (10)。10 更小，80 > 10 → 80 下沉到 5，10 上浮到 2。
- 80 在位置 5，儿子：10 (90), 11 (60)。60 更小，80 > 60 → 80 下沉到 11，60 上浮到 5。
- 80 在位置 11 无儿子，停止。

**对 1 (值为 150) 做 PercolateDown：**
- 儿子：2 (10), 3 (40)。10 更小，150 > 10 → 150 下沉到 2，10 上浮到 1。
- 150 在位置 2，儿子：4 (20), 5 (60)。20 更小，150 > 20 → 150 下沉到 4，20 上浮到 2。
- 150 在位置 4，儿子：8 (100), 9 (30)。30 更小，150 > 30 → 150 下沉到 9，30 上浮到 4。
- 150 在位置 9 无儿子，停止。

```
最终的最小堆：
               10
           /       \
         20         40
       /    \      /    \
     30     60   50     110
    /  \   /  \  /  \   /  \
  100 150 90 80 70 120 140 130
```

### 5.4 为什么是 $O(N)$？

!!! abstract "定理"
    对于高度为 $h$、包含 $2^{h+1}-1$ 个节点的完美二叉树，所有节点的高度之和为 $2^{h+1} - 1 - (h+1)$。

**直观理解：** 大部分节点都在底层（叶节点），下沉距离很短；只有少数节点在高层，下沉距离长。

建堆的总时间 = 每个节点的下沉距离之和 $\propto N$，所以复杂度为 $O(N)$。

!!! tip "跟逐个 Insert 对比"
    | 方法 | 复杂度 |
    |------|--------|
    | 逐个 Insert | $O(N\log N)$ |
    | BuildHeap（一次性） | $O(N)$ |

## 6. d-堆

所有节点有 $d$ 个儿子的堆。

- **好处：** 树更矮 → 插入时上浮距离更短
- **坏处：** DeleteMin 需要 $d-1$ 次比较来找最小儿子 → $O(d\log_d N)$
- **适用场景：** 优先队列太大、主存放不下时（例如外部排序）

---

## 7. 优先队列的应用

!!! example "第 $k$ 大元素"
    给 $N$ 个元素，找第 $k$ 大的。
    
    - 方法一：排序后取 $A[k]$ → $O(N\log N)$
    - 方法二：建大小为 $k$ 的最小堆 → 遍历剩余元素，比堆顶大就替换 → $O(N\log k)$
