## 1. Public Key Cryptography（公钥密码学）

### 1.1 背景

传统密码学（Private-key protocols）：Alice 和 Bob 需要**提前见面**，商量好一个密码本（codebook），之后用这个密码本加密所有通信。Eve（窃听者）只能靠收集加密消息来尝试破解密码本。

**问题：** 如果 Alice 和 Bob 从未见过面，怎么安全通信？

### 1.2 RSA 的核心思想

RSA（1970年代）：Bob 可以实现一个**数字锁**，只有他自己有钥匙。Bob 把这个锁公开（公钥），任何人（包括 Alice）都可以用它来锁住消息，但只有 Bob 能打开（私钥）。

**角色：**
- Alice：发送消息的人
- Bob：接收消息的人
- Eve：窃听者，能看到所有加密消息但解不开

每个人有一对密钥：
- **公钥 (public key)：** 全世界都知道，用来加密
- **私钥 (private key)：** 只有自己知道，用来解密

## 2. RSA 方案详解

### 2.1 密钥生成（Bob 的操作）

1. 选两个大质数 $p$ 和 $q$（通常各 512 位）
2. 令 $N = pq$
3. 选 $e$ 使得 $\gcd(e, (p-1)(q-1)) = 1$（通常选 $e = 3$）
4. 算 $d = e^{-1} \pmod{(p-1)(q-1)}$（即 $ed \equiv 1 \pmod{(p-1)(q-1)}$）

**公钥：** $(e, N)$ —— 发布给全世界

**私钥：** $d$ —— 只有自己知道

**关键：** $p$ 和 $q$ **不能公开**！如果 Eve 知道了 $p$ 和 $q$，她就能算出 $d$。

### 2.2 加密和解密

**加密（Alice 的操作）：**

$$E(x) = x^e \pmod N$$

其中 $x$ 是要发送的消息（看作 mod $N$ 下的整数，排除 0 和 1）。

**解密（Bob 的操作）：**

$$D(y) = y^d \pmod N$$

这会恢复原始消息 $x$，即 $D(E(x)) = x$。

### 2.3 示例

选 $p = 5, q = 11$，则 $N = pq = 55$。

$(p-1)(q-1) = 4 \times 10 = 40$。

选 $e = 3$，$\gcd(3, 40) = 1$。公钥：$(3, 55)$。

求 $d = 3^{-1} \pmod{40} = 27$（因为 $3 \times 27 = 81 \equiv 1 \pmod{40}$）。私钥：$27$。

**发送消息 $x = 13$：**

- 加密：$y = 13^3 = 2197 \equiv 52 \pmod{55}$
- 解密：$52^{27} \pmod{55} = 13$（恢复原始消息）

## 3. RSA 正确性证明

### 3.1 定理

$$D(E(x)) = (x^e)^d \equiv x \pmod N \quad \forall x \in \{0, 1, ..., N-1\}$$

### 3.2 证明（利用费马小定理 FLT）

由 $d$ 的定义：$ed \equiv 1 \pmod{(p-1)(q-1)}$，即存在整数 $k$ 使得：

$$ed = 1 + k(p-1)(q-1)$$

要证 $x^{ed} \equiv x \pmod N$，即证 $x^{ed} - x \equiv 0 \pmod N$。

$$x^{ed} - x = x^{1+k(p-1)(q-1)} - x = x(x^{k(p-1)(q-1)} - 1)$$

**证明这个式子能被 $p$ 整除：**

- **Case 1：$x$ 不是 $p$ 的倍数。** 则 $x 
ot\equiv 0 \pmod p$，由 FLT 得 $x^{p-1} \equiv 1 \pmod p$，所以 $x^{k(p-1)(q-1)} - 1 \equiv 0 \pmod p$。
- **Case 2：$x$ 是 $p$ 的倍数。** 则式子中 $x$ 本身就有因子 $p$，显然能被 $p$ 整除。

**同理**这个式子也能被 $q$ 整除。

因为 $p$ 和 $q$ 都是质数，所以这个式子能被 $pq = N$ 整除。即 $x^{ed} \equiv x \pmod N$。

### 3.3 替代证明（用中国剩余定理 CRT）

先用 FLT：

$$x^{ed} = x^{k(p-1)(q-1)+1} = (x^{k(q-1)})^{p-1} \cdot x \equiv x \pmod p$$

（当 $x 
ot\equiv 0 \pmod p$ 时由 FLT 得证；当 $x \equiv 0 \pmod p$ 时显然成立。）

同理：$x^{ed} \equiv x \pmod q$。

由 CRT，$x^{ed} \equiv x \pmod p$ 且 $x^{ed} \equiv x \pmod q$，且 $\gcd(p, q) = 1$，则在 mod $N$ 下有**唯一解**，这个解就是 $x$。所以 $x^{ed} \equiv x \pmod N$。

## 4. RSA 安全性

### 4.1 基本假设

> 给定 $N, e$ 和 $y = x^e \pmod N$，**没有高效的算法**能求出 $x$。

### 4.2 Eve 的攻击方法

**方法 1：暴力枚举**

尝试所有可能的 $x$，逐一检查 $x^e \equiv y \pmod N$。但 $x$ 的范围是 $N$（512 位 = $2^{512}$ 种可能），完全不现实。

**方法 2：分解 $N$**

如果 Eve 能把 $N$ 分解成 $p \times q$，她就能算出 $(p-1)(q-1)$，然后用扩展欧几里得算出 $d = e^{-1} \pmod{(p-1)(q-1)}$，从而解密所有消息。

但**大整数分解**被认为是极难的问题——没有已知的高效算法。

### 4.3 重要注意

RSA 的安全性**没有被严格证明**。它依赖于两个假设：
- 破解 RSA 本质上等价于分解 $N$
- 大整数分解是困难的

---

## 5. 实现问题

### 5.1 如何找到大质数？

**素性测试（Primality Testing）：** 给定一个正整数 $n$，存在高效算法能判断 $n$ 是否为质数。

- 素性测试的时间复杂度：$O((\log n)^k)$，即 $n$ 的位数的低次多项式
- **注意：** 我们能高效判断一个数是不是质数，但如果是合数，我们**不能**高效找到它的因子！RSA 的安全性正依赖这个区别。

**实际操作：** Bob 随机生成 512 位整数，逐个做素性测试，直到找到两个质数。

### 5.2 Prime Number Theorem（素数定理）

$$\pi(n) \geq \frac{n}{\ln n} \quad (n \geq 17)$$

其中 $\pi(n)$ 表示小于等于 $n$ 的质数个数。

**在 RSA 中的意义：**

当 $n = 2^{512}$ 时，大约每 355 个 512 位整数中就有一个是质数。所以平均只需要测试约 355 个数就能找到一个质数，效率很高。

### 5.3 模幂运算（Modular Exponentiation）

Alice 需要计算 $x^e \pmod N$，Bob 需要计算 $y^d \pmod N$。

用**快速幂（Repeated Squaring）**算法，乘法次数为 $O(\log N)$。所有中间结果都在 mod $N$ 下，所以数字位数不超过 $O(\log N)$。

总时间复杂度：$O((\log N)^3)$。

---

## 6. 总结

| 操作 | 谁做 | 计算 | 复杂度 |
|------|------|------|--------|
| **密钥生成** | Bob | 找质数 $p, q$，算 $d$ | $O((\log N)^3)$ |
| **加密** | Alice | $x^e \pmod N$ | $O((\log N)^3)$ |
| **解密** | Bob | $y^d \pmod N$ | $O((\log N)^3)$ |
| **破解（Eve）** | Eve | 分解 $N = pq$ | 极难（无高效算法） |

Alice 和 Bob 只需要做简单的乘法、幂运算和素性测试——口袋计算器都能搞定。Eve 要破解则需要分解大整数——以目前全人类的算力加起来也做不到。这就是 RSA 革命性之处。
