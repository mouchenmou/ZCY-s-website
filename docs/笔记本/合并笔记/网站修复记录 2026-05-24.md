# 网站修复记录 2026-05-24

## 问题 1：侧栏 navigation.indexes 肥大 + 双按钮

### 现象
给课程加了 `- 笔记本/CS61A/index.md` 等 index 条目并启用 `navigation.indexes` 后，侧栏里带 index 的课程（CS61A、CS61B 等）出现：
- 显示成**肥大卡片**样式
- **两个按钮**：课程名一个按钮，`>` 箭头一个按钮
- 而不带 index 的课程（流体力学、大学物理实验）是正常的简洁单行样式

### 根因
1. **Material 9.7 默认模板**对 `navigation.indexes` 的渲染方式与旧版不同，会渲染成独立的卡片样式
2. **CSS 冲突**：`extra.css` 第 1452 行有通用规则 `[data-md-color-scheme="warm"] .md-nav__link { padding: 12px 16px !important }`，同样作用于 `.md-nav__link--index`，导致 div 被撑高

### 解决方案

#### （1）自定义 nav-item.html 模板
参照 BruceJqs 的 `MyNotebook` 仓库，创建了 `overrides/partials/nav-item.html`，将带 index 的 section 渲染成：
```html
<div class="md-nav__link md-nav__link--index">
  <a href="index页">课程名</a>
  <label>▸</label>
</div>
```
一个干净的单行 div。

#### （2）mkdocs.yml 改动
- 在 `theme` 下添加了 `custom_dir: overrides`
- 启用了 `navigation.indexes` feature

#### （3）extra.css 改动
- 给 `.md-nav__link--index` 添加了 `padding: 0`、`background: transparent`、`border-radius: 0` 等规则，彻底扒掉 Material 默认的卡片样式
- 将通用导航链接规则从 `.md-nav__link` 改为 `.md-nav__link:not(.md-nav__link--index)`，从根源上排除冲突

### 最终效果
侧栏里带 index 的课程和不带 index 的课程外观完全一致——单行、无背景、无额外 padding。

## 问题 2：代码块行号和代码错位、行号列独立圆角框

### 现象
- 代码行号和代码文字**错位不对齐**
- 行号列有自己的**圆角白底框**，和代码列分离
- 跟 BruceJqs 的代码块渲染（同一灰框 + 灰色分隔线）差距很大

### 解决方案（extra.css 改动）
- 删除了 `color: #2d3748 !important`（之前把所有代码颜色强制成了灰色，导致语法高亮失效）
- 删除了 `.md-typeset pre { font-size: 0.88em }`（代码文字被缩小但行号没变，导致错位）
- 给行号列（`.linenos`、`.linenodiv` 等）去掉独立背景和圆角
- 整体代码块容器保持 `border-radius: 12px`、灰色背景 `#f5f5f5`，行号和代码在同一个框内

## 问题 3：PDF 嵌入内联

### 尝试
- 参照 Slowist-Lee 的 Jekyll 站点，尝试用 `<object>` 标签在 md 文件中嵌入 PDF
- 为 CS70、流体力学、信号与系统等课程的 PDF 创建了 index.md 包装文件

### 结果
- **信号与系统的章节部分**渲染成功
- **其他课程全部失败**，PDF 只能显示空白的 object 框
- 且显示在主页上的 PDF 如果字号太小难以阅读

### 最终决定
**放弃 PDF 嵌入**，恢复为直接跳转到 PDF 文件的链接方式。所有改动已回退。

> 注意：如果将来再次尝试 PDF 嵌入，信号与系统的成功案例值得研究——可能是因为该目录已经在 nav 中建立了正确的嵌套结构。

## 问题 4：移动端导航抽屉无法点击（三个杠）

### 现象
缩小浏览器窗口后，左上角三个杠（汉堡菜单）点开后无法点击任何导航项。

### 根因
`extra.css` 中 `.md-content` 的 `box-shadow: -8px 0 25px ..., 8px 0 25px ...` 向左右延伸，覆盖了导航抽屉区域，同时 `z-index: 2` 把内容区层级抬高了。

### 解决方案
- 去掉左右方向的 box-shadow，只保留向下阴影
- 去掉 `.md-content` 的 `z-index: 2`
- 给 `.md-nav--primary` 加 `z-index: 3`
