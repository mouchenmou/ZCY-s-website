

我原本的个人网站本地文件夹是放在Windows的桌面上的，但是每次背着Windows出门做笔记真的心累。所以在Mac上clone了一份，每次出门都在Mac上改动笔记，晚上回寝室的时候用Windows执行一下  `git pull` 就好了。不过需要注意的是，Obsidian中的插件不会被同步过来，因为它根本就没有被上传到Github中，并且不建议同步插件，因为不同系统中插件的路径一般不兼容。

因此我又重新装了Terminal插件和Claudian插件。Terminal插件直接去插件市场找一下就好了。

Claudine插件在插件市场中是找不到的，需要自己去安装，直接把插件地址`https://github.com/YishenTu/claudian`丢给Claude让它帮你装就行了。只不过在Terminal插件中调教的时候发现，终端还是没有跟着系统的代理走，所以一直clone失败，让Terminal中的claude帮你配置一个临时代理之后再进行clone就行

| 操作                                 | 命令                                           | 操作的主机   |
| ---------------------------------- | -------------------------------------------- | ------- |
| 在MacBook上改动笔记                      | `git add . → git commit -m "xxx" → git push` | Macbook |
| 在MacBook上改动笔记后，<br />在Windows上接收改动 | `git stash → git pull`                       | Windows |
| 在Windows上改动笔记                      | `git add . → git commit -m "xxx" → git push` | Windows |
| 在Windows上改动笔记后，<br />在MacBook上接收改动 | `git stash → git pull`                       | MacBook |

下面会介绍为什么要先执行`git stash`。

## 一、超级重要的使用事项：

我之前在Windows中改动了一个笔记并git push之后，在Mac上git pull失败了。报错如下：

```Unix
  ZCY-s-website % git pull
  remote: Enumerating objects: 31, done.
  remote: Counting objects: 100% (31/31), done.
  remote: Compressing objects: 100% (5/5), done.
  remote: Total 18 (delta 10), reused 17 (delta 10), pack-reused 0 (from 0)
  Unpacking objects: 100% (18/18), 11.38 KiB | 237.00 KiB/s, done.
  From github.com:mouchenmou/ZCY-s-website
     79eefb6d..0de51409  main       -> origin/main
     c4d3741b..63ff3d37  gh-pages   -> origin/gh-pages
  Updating 79eefb6d..0de51409
  error: Your local changes to the following files would be overwritten by merge:
          docs/笔记本/.obsidian/workspace.json
  Please commit your changes or stash them before you merge.
  Aborting
```
### 报错原因：
我在MacBook中执行`git pull`的时候，Mac本地的Obsidian处于打开状态，它自动修改了 `workspace.json`（记录窗口状态的），`git`发现这个文件你本地有改动，怕被覆盖所以不让你 pull。因此失败了。

### 解决方法：

1. 先运行`git stash`
2. 再运行`git pull`

`git stash` 会把本地的临时改动先藏起来，`git pull` 拉取远程改动。之后就能正常用了。

鉴于我这个人习惯一直开着Obsidian，所以以后每次执行`git pull`前都应该先执行`git stash`。哪怕Obsidian处于关闭状态，执行`git stash`也不会有任何影响，只会给你返回一个`No lacal changes to save`，所以还是很安全的。

## 二、配置过程的注意事项：

不是说在Mac那边git clone之后就能用的，还要进行以下操作：

### 1. SSH 配置（身份认证）

  SSH用于向 GitHub 证明"你是你"，这样 git push 时就不需要每次输入密码或 Token。

1. 生成密钥

  `ssh-keygen -t ed25519 -C "你的GitHub邮箱"`

  注意： 命令中所有参数都是连续的，不要换行。一路按回车即可（不用设密码）。

2. 查看公钥

  `cat ~/.ssh/id_ed25519.pub`


3. 添加到 GitHub

4. 复制上面输出的一整行内容（以 ssh-ed25519 开头）
5. 打开 https://github.com/settings/keys
6. 点击 New SSH key
7. Title 随便填（比如 MacBook），Key 粘贴进去
8. 点击 Add SSH key

9. 切换远程地址为 SSH

  默认 clone 下来的是 HTTPS 地址，需要改成 SSH：

  git remote set-url origin git@github.com:用户名/仓库名.git

  验证是否切换成功：

  git remote -v

  应该看到 git@github.com: 开头的地址。

10. 首次连接确认指纹

  第一次 git pull 或 git push 时会提示：

  The authenticity of host 'github.com' can't be established.
  Are you sure you want to continue connecting (yes/no/[fingerprint])?

  输入 yes 回车。这是 GitHub 的官方指纹，确认一次后永久信任。

  ---
### 2. 代理配置（网络访问）

代理解决的是"能不能访问到 GitHub"的问题。校园网或国内网络通常需要挂代理才能连接 GitHub。

1. 设置代理

  先确认你的代理端口是多少（Clash 默认 7890，其他代理可能不同）。

  `git config --global http.proxy http://127.0.0.1:你的端口号`
  `git config --global https.proxy http://127.0.0.1:你的端口号`


2. 清除代理

  **如果去了美国或其他不需要代理的网络环境，就要执行以下操作：**

  `git config --global --unset http.proxy`
  `git config --global --unset https.proxy`

3. 查看当前代理配置

  `git config --global --list | grep proxy`

4. 代理配置是永久的

  git config --global 写入的是 ~/.gitconfig 文件，配置一次永久生效。不需要每次新开终端都重新设置。它只对 git
  命令有效，不影响终端里其他命令的网络。

5. 回国之后还得重新执行步骤一把代理改回来

## 三、过程中踩到的坑

两台电脑不要同时改同一个文件，我就是在Mac上`git clone`了整个文件之后，在Windows上又改动了一个文件，并且`git push`了。结果我还没用Mac把整个改动`git pull`下来我就在Mac中对同一个文件进行了改动。这就导致两个文件产生冲突了，还得`Resolve in merge editor`。所以要养成一个好习惯，就是每次在一台设备上改动后，另一台设备一定要及时`git pull`，然后再进行后续的改动。
