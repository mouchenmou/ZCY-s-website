## Graph Definition
A simple graph is a graph with:

- No edges that connect a vertex to itself（No loops）
- No two edges that connect the same vertices（No parallel edges）

## Graph Terminology
- Graph:
    - Set of **vertices**, a.k.a. nodes.
    - Set of **edges**: Pairs of vertices
    - Vertices with an edge between are **adjacent**
    - Optional: Vertices or edges may have **weights (or labels)**
- A **path** is a sequence of vertices connected by edges
- A **cycle** is a path whose first and last vertices are the same.
    - A graph with a cycle is "cyclic"
- Two vertices are **connected** if there is a path between them. If all vertices are connected, we say the graph is connected.

### Depth First Traversal
This idea of exploring a neighbour's entire subgraph before moving on to the next neighbour is known as Depth First Traversal.

- Example: Explore 1's subgraph completely before using the edge 0-3.
- Called "depth first" because you go as deep as possible.

![](附件/Pasted%20image%2020260501230759.png)

#### DFS PostOrder
我们刚刚演示的DepthFirstPaths算法称作 **DFS PreOrder**，我们同样可以执行 **DFS PostOrder** 

- **DFS PostOrder：** Action is afer DFS calls neighbour
- Example: dfs(s):
    - mark(s)
    - For each unmarked neighbour n of s, dfs(n)
    - print(s)
- Result for dfs(0) would be: **347685210**

#### BFS order（广度优先搜索）
BFS order: Act in order of distance from s

- BFS stands for "breadth first search"
- Analogous to "level order". Search is wide, not deep
- Result would be: **0 1 24 53 68 7**
