## 3. Arrays vs. Classes
![](附件/Pasted%20image%2020260312232910.png)

!!! Explanation "## Array indices can be computed at runtime but Classes cannot"
    Array indices can be computed at runtime.
    ![](附件/Pasted%20image%2020260312233130.png)
    Classes member variable names CANNOT be computed and used at runtime.
    ![](附件/Pasted%20image%2020260312233350.png)
    Dot notation does't work either。
    ![](附件/Pasted%20image%2020260312233502.png)

The only way to access a member of a class is with hard-coded dot notation.
![](附件/Pasted%20image%2020260312233604.png)
### 💡 终极总结（也就是最后一张图那句话）

**The Java compiler does not treat text on either side of a dot as an expression.** _(Java 编译器不会把点 `.` 两边的东西当成需要计算的表达式。)_

- **对于数组：** `[]` 里面可以放变量、放算式（比如 `x[i + 1]`），程序运行的时候会去算出具体的值。
    
- **对于类：** `.` 的右边**只能放实打实的、写死的属性名**（比如 `p.mass`）。它不能是变量，不能被动态替换。
    

用个通俗的比喻：

- **数组**就像是酒店的房间号。你可以打车跟司机说：“带我去客人告诉我的那个房间号”（动态）。
    
- **类**就像是你家的厨房、卧室。你不能跟装修师傅说：“给我修一个名字叫做 `String roomName = "厨房"` 的房间”。你必须在图纸上白纸黑字写死：“这里是厨房”（Hard-coded）。
