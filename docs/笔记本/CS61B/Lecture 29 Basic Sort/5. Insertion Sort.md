### 5.1 基本思路

就像打牌时**摸牌**：

- 手里已经有一把排好序的牌
- 每摸一张新牌，从右往左找到它该在的位置
- 比它大的牌全部右移一位，然后把新牌塞进去

### 5.2 In-place 版本

[这是In-place Insertion Sort的完整过程](https://docs.google.com/presentation/d/10b9aRqpGJu8pUk8OpfqUIEEm8ou-zmmC7b_BE5wgNg0/edit?slide=id.g463de7561_042#slide=id.g463de7561_042)

用**交换**代替整体右移：

```
P O T A T O      ← 初始
P O T A T O      (0 次交换)  ← P 不动
O P T A T O      (1 次交换)  ← O 插到 P 前面
O P T A T O      (0 次交换)  ← T 已经在正确位置
A O P T T O      (3 次交换)  ← A 一路交换到最前面
A O P T T O      (0 次交换)
A O O P T T      (3 次交换)
───────────────
总计：7 次交换
```

!!! tip "代码本质上就是"
    ```java
    for (int i = 1; i < N; i++) {
        for (int j = i; j > 0 && a[j] < a[j-1]; j--) {
            swap(a, j, j-1);
        }
    }
    ```

### 5.3 复杂度

| | |
|---|---|
| 最好 | $\Theta(N)$（数组已经有序，一次交换都不需要） |
| 最坏 | $\Theta(N^2)$（逆序的数组） |
| 空间 | $\Theta(1)$ |

!!! warning "Josh 的严谨补充"
    Josh 说技术上 $\Omega$ 不一定是"最好情况"——$\Omega(1)$ 也是对的（因为至少要做常数次操作）。但我们习惯用 $\Omega(N)$ 描述最好情况的量级。

### 5.4 插入排序的甜蜜点

!!! abstract "两个场景下插入排序是最快的"
    1. **几乎有序的数组**（逆序对数量 $\leq cN$）
       - 每次交换消除恰好一个逆序对
       - 运行时间 $\Theta(N + K)$，$K$ 是逆序对数量
       - 对于几乎有序的数组 → $\Theta(N)$！
    
    2. **小数组**（$N < 15$ 左右）
       - 这是实验结论而非理论结论
       - 粗略解释：归并/堆排花太多时间在"分"上，插入排序直接"治"
       - Java 的归并排序实现里就用了这个技巧：小数组切成插入排序

!!! example "经典面试题：选哪个排序？"
    > 读取 100 万个整数 → 归并排序 → 随机改一个数 → 再排序。用哪个算法最快？
    
    **答案：插入排序。**
    
    归并排完之后改一个数，最多产生 999,999 个逆序对（$\Theta(N)$）。插入排序在几乎有序数组上的运行时间是 $\Theta(N)$——把另外三个 $O(N \log N)$ 甚至 $O(N^2)$ 的算法全秒了。
