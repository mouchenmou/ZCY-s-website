### 3.1 Naive Heapsort

Selection Sort 每次都要扫一遍整个数组找最小元素，这太蠢了。 **堆可以帮我们在 $O(1)$ 时间拿到最大/最小元素** ——这正是我们需要的。

!!! tip "为什么用大顶堆？"
    后面会看到，用大顶堆的话，每次把堆顶（最大值）换到数组末尾，堆的大小就自然缩 1 了。 **如果用最小堆就没法原地做了。 **

[这是Naive Heapsort的完整过程](https://docs.google.com/presentation/d/1HVteFyWOxBW4mmUgkDnpUoTkWexiHt7Ei30Qolbc_I4/edit?slide=id.g463de7561_042#slide=id.g463de7561_042)，看完之后就懂下面的逻辑了

```
Naive Heapsort：
1. 把所有元素插入大顶堆，丢弃原始数组，创建一个新输出数组
2. 重复 N 次：
   a. 从大顶堆中删除最大元素 —— O(log N)
   b. 把最大元素放到输出数组的末尾（未使用部分）
```

**复杂度分析：**

- 建堆（N 次插入）：$O(N \log N)$
- 选最大（N 次）：$\Theta(1)$
- 删最大（N 次）：$O(N \log N)$
- **总时间：$O(N \log N)$** ——比 Selection Sort 的 $\Theta(N^2)$ 好太多了

但 **空间是 $\Theta(N)$** ——多复制了一份数据。比 Selection Sort 的 $\Theta(1)$ 差，不过一般来说问题不大。能不能既 $O(N \log N)$ 又快又省空间？这就是下一节要讲的。

---

### 3.2 In-place Heapsort（原地堆排序）

!!! explanation "核心 trick"
    **不建新数组——直接把输入数组当成堆来用！**
    
    怎么把一个乱七八糟的数组变成堆？方法叫 **bottom-up heapification**（自底向上堆化）：按**逆层序**（reverse level order），对每个节点做**sink**操作。

这里下标 0 不放空（跟之前堆的实现不一样），直接用。

[这是In-place Heapsort的完整过程](https://docs.google.com/presentation/d/1SzcQC48OB9agStD0dFRgccU-tyjD6m3esrSC-GLxmNc/edit?slide=id.g463de7561_042#slide=id.g463de7561_042) 

```
原始数组： [32, 15, 2, 17, 19, 26, 41, 17, 17]
              ↓ bottom-up heapify（逆层序 sink）
堆化后：   [41, 19, 32, 17, 15, 26, 2, 17, 17]
              ↓ 不停把堆顶（最大）换到末尾 + sink 新堆顶
最终：     已排序！
```


**复杂度：**

| 步骤 | 时间 |
|------|------|
| Bottom-up heapify | $O(N \log N)$（N 次 sink，每次最多 $\log N$） |
| 选最大（N 次） | $\Theta(1)$ |
| 删最大（N 次） | $O(\log N)$ |
| **总计** | **$O(N \log N)$** |

!!! tip "bottom-up heapify 其实只要 $\Theta(N)$"
    虽然直观上看 N 次 sink 每次 $O(\log N)$ 就是 $O(N \log N)$，但实际上大部分节点在底层，sink 不了几步。更精确的分析可以证明 bottom-up heapify 最坏情况是 $\Theta(N)$。不过这不影响最终结论——堆排序还是 $\Theta(N \log N)$，因为后面 N 次删最大已经 $\Theta(N \log N)$ 了。

**空间复杂度：$\Theta(1)$** ——原地完成！不需要任何额外数组。

!!! warning "一个小注释"
    如果堆操作用递归实现，空间是 $\Theta(\log N)$（递归调用栈）。但 $\Theta(\log N)$ 和 $\Theta(1)$ 在实际中基本没区别。

### 3.3 一些值得注意的细节

#### 最佳情况：全是重复元素 → $\Theta(N)$

如果数组里的元素 **全部相同** （比如全是 17），每次 sink 操作根本不用动，直接到底。堆排序在这种情况下只需要 $\Theta(N)$ 时间。

#### 缓存性能差

这是堆排序的一个实际短板（61C 会详细讲）。因为堆排序在数组里跳来跳去（父子节点下标差很远）， **CPU 缓存命中率低** 。同样的 $O(N \log N)$，Mergesort 在实际中往往比堆排序快。
