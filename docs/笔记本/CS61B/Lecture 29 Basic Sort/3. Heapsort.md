### 3.1 Naive Heapsort

!!! tip "核心想法"
    用堆来维护最小值，而不是每次扫整个数组！

这里我们用**大顶堆**（max-oriented heap）——原因马上会看到。

```
Naive Heapsort：
1. 把所有元素插入大顶堆，丢弃原始数组，创建一个新输出数组
2. 重复 N 次：
   a. 从大顶堆中删除最大元素
   b. 把最大元素放到输出数组的末尾（未使用部分）
```

**复杂度分析：**
- 建堆：$O(N \log N)$
- 选最大：$\Theta(1)$ × N 次
- 删最大：$O(\log N)$ × N 次
- **总时间：$O(N \log N)$**

但**空间是 $\Theta(N)$**——多复制了一份数据。

### 3.2 In-place Heapsort（原地堆排序）

!!! explanation "原地堆化的巧妙之处"
    不建新数组——**直接把输入数组当成堆来用**！

    方法叫**bottom-up heapification**（自底向上堆化）：按**逆层序**依次下沉（sink）每个节点。

    > 注意这里下标 0 不放空，直接用。

```
原始数组：[32, 15, 2, 17, 19, 26, 41, 17, 17]
           ↓ bottom-up heapify
堆化后：  [41, 19, 32, 17, 15, 26, 2, 17, 17]
           ↓ 不停把堆顶（最大）换到末尾 + sink
最终：    已排序！
```

**复杂度：**

| 步骤 | 时间 |
|------|------|
| Bottom-up heapify | $O(N \log N)$（专家可证 $\Theta(N)$） |
| 选最大（N 次） | $\Theta(1)$ 每次 |
| 删最大（N 次） | $O(\log N)$ 每次 |
| **总计**|**$O(N \log N)$** |

**空间复杂度：$\Theta(1)$**——原地完成！

!!! warning "一个小注释"
    如果堆操作用递归实现，空间是 $\Theta(\log N)$。但 $\Theta(\log N)$ 和 $\Theta(1)$ 在实际中基本没区别。
