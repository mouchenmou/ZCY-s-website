## 1.1 什么是排序？

!!! abstract "排序的定义"
    对于一个序列，**排序**就是把它重排成**非递减**顺序：$x_1 \leq x_2 \leq \cdots \leq x_N$。

排序依赖一个**顺序关系**，它必须满足两条性质：

| 性质 | 含义 |
|------|------|
| **三分律** | $a < b$、$a = b$、$b < a$ 三者恰好一个成立 |
| **传递律** | 若 $a < b$ 且 $b < c$，则 $a < c$ |

满足这两条的关系也叫**全序（total order）**。

!!! example "例子：按字符串长度排序"
    对 `["cows", "get", "going", "the"]` 按长度排序：
    - `["the", "get", "cows", "going"]` ✓
    - `["get", "the", "cows", "going"]` ✓
    
    注意：在这个关系下，`len("the") = len("get")`，所以 "the" 和 "get" 被视为 **=**（相等）。
    
    > 这个 `=` 不是 Java 里的 `.equals()`——它是排序关系下的"等价"。

## 1.2 逆序对视角

!!! conception "逆序对（Inversion）"
    一对**顺序搞反了**的元素。
    
    另一个角度看排序：你有一个包含 $Z$ 个逆序对的序列，排序就是**执行一系列操作把 $Z$ 减到 0**。

```
0 1 1 2 3 4 8 6 9 5 7
              ↑ ↑ ↑ ↑ ↑
              8-6, 8-5, 8-7, 6-5, 9-5, 9-7
              
              6 个逆序对（最多 55 个）
```

## 五种排序算法总结

|                   | 最好                 | 最坏                 | 空间          | 备注             |
| ----------------- | ------------------ | ------------------ | ----------- | -------------- |
| Selection Sort    | $\Theta(N^2)$      | $\Theta(N^2)$      | $\Theta(1)$ | 简单但笨           |
| Heapsort（Inplace） | $\Theta(N)^*$      | $\Theta(N \log N)$ | $\Theta(1)$ | Cache 性能差      |
| Mergesort         | $\Theta(N \log N)$ | $\Theta(N \log N)$ | $\Theta(N)$ | 这些里最快的         |
| Insertion Sort    | $\Theta(N)$        | $\Theta(N^2)$      | $\Theta(1)$ | **小数组/几乎有序**最优 |
| Shell's Sort      | $\Theta(N)$        | $\Omega(N \log N)$ | $\Theta(1)$ | 理论丰富           |

*全部重复元素的数组给堆排序带来线性最好情况。
