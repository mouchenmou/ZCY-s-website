!!! abstract
    LP = 线代 + 不等式 + 最小化。$Ax=b$ 且 $x\ge 0$ 下最小化 $c^T x$。可行集合是凸多面体——最佳解必在顶点上。单纯形法从一个顶点沿边走到另一个。原始和对偶问题的最优值相等。

---

## 1. LP 是什么

LP 是线代加上两个新概念：不等式和最小化。初始条件还是矩阵等式 $Ax=b$，但现在解必须是非负的——$x \ge 0$，即 x 的分量都不能是负数。矩阵的形状是 $n>m$——未知数多于方程个数，所以可能会存在很多 $x \ge 0$ 满足 $Ax=b$。LP 选择的解 $x^* \ge 0$ 是代价最小的：

> 代价定义为 $c_1 x_1 + \cdots + c_n x_n$，此时这个非负向量 $x^*$ 是 $Ax=b$ 的解，并且具有最小的代价。

LP 问题可以从一个矩阵 A 和两个向量 b、c 开始：
1. A 的 $n>m$，如 $A = [1,1,2]$，3 个未知数、1 个方程
2. b 是 m 个分量的向量，如 $b = [4]$
3. 费用向量 c 有 n 个分量，如 $c = [5,3,8]$

**目标：** 以 $Ax=b$ 和 $x\ge 0$ 为前提，最小化 $c \cdot x$。如：限制在 $x_1+x_2+2x_3=4$，$x_1,x_2,x_3 \ge 0$ 的情况下，最小化 $5x_1+3x_2+8x_3$。

以上解释了 LP 是怎么回事，但它到底是怎么来的呢？实际上，LP 是数学对管理的最重要应用之一。因为有 $x^* \ge 0$ 和最小化 $c^T x^*$ 的要求，寻找 $x^*$ 比求解 $Ax=b$ 更难。下面介绍著名的**单纯形法**和**内点法**。

## 2. 可行集合——三角形 PQR

首先看一下限制条件 $Ax=b$ 和 $x\ge 0$。方程 $x_1+x_2+2x_3=4$ 给出的是三维下的一个平面，而 $x_1,x_2,x_3 \ge 0$ 的要求把这个平面限制为三角形 PQR。最佳解 $x^*$ 必须在三角形 PQR 之上。

![](附件/image-20210713142826566.png)

这个三角形包含了所有满足 $Ax=b$ 和 $x\ge 0$ 的向量 x，这些 x 称为**可行点**，三角形称为**可行集合**。这些点都是可能最小化 $c \cdot x$ 的候选。LP 的最后一步：**在 PQR 三角形上，求最小化 $5x_1+3x_2+8x_3$ 的解 $x^*$。**

在平面 $5x_1+3x_2+8x_3=0$ 上，向量的耗费都为 0。但注意这个平面并没有和三角形相交，所以不可能取得 0 耗费的解。所以不断增大 C，平面 $5x_1+3x_2+8x_3=C$ **平行地**向三角形移动。

第 1 个和三角形接触到的平面就拥有最小耗费 C（最小耗费就是 C！），而且它接触到三角形的点就是最佳解 $x^*$。**这个点必须是三角形的顶点 P、Q 或 R——因为不能在不接触到顶点的情况下，首先接触到三角形的内部！**

$$P = (4,0,0) \text{ costs } 20, \quad Q = (0,4,0) \text{ costs } 12, \quad R = (0,0,2) \text{ costs } 16$$

Q 胜出了，所以 $x^* = (0,4,0)$ 是这个 LP 问题的解。

如果耗费向量 c 改变了，那么不断移动的平行平面的倾斜角度就不一样了。如果 c 的改变很小，那么 Q 还是胜出点。但如果改变幅度比较大，如 $c \cdot x = 5x_1+4x_2+7x_3$，最佳的 $x^*$ 变成 $R = (0,0,2)$，此时最小耗费是 $7 \times 2 = 14$。

**要点 1：** 一些线性程序会**最大化利润**，而不是最小化代价。这在数学上是等价的。只是现在，平行平面以很大的 C 开始（而不是很小的 c），不断的减小，从而平面不断向原点逼近（而不是远离），当 C 越来越小，第 1 个接触到的点仍然是三角形的顶点。

**要点 2：** 有时候 $Ax=b$ 和 $x\ge 0$ 是不可能被满足的。如方程 $x_1+x_2+x_3 = -1$ 不可能在 $x \ge 0$ 的限制条件下求解——此时可行集合是**空集**。

**要点 3：** 可行集合也有可能是**无界**的。如果 $Ax=b$ 是 $x_1+x_2-2x_3=4$，那么正向量 $(100,100,98)$ 是一个候选，更大的正向量 $(1000,1000,998)$ 同样也是。此时可行集合不再被限制为一个三角形区域——虽然 P, Q 顶点还是候选，但 R 顶点相当于处在无穷的位置。

**要点 4：** 如果可行集合是无界的，那么最小代价可以是 $-\infty$。如假设耗费方程是 $-x_1-x_2+x_3$，那么向量 $(100,100,98)$ 的耗费是 $C=-102$，向量 $(1000,1000,998)$ 的耗费是 $C=-1002$。这相当于变量 $x_1, x_2$ 会帮助我们减小耗费，而不是增加。虽然现实情况下不会发生，但理论上，A、b、c 三者合起来可以产生难以预期的可行集合和耗费。

## 3. 原始问题和对偶问题

以 Fig8.6 的问题开始。3 个未知数 $x_1,x_2,x_3$ 分别代表研究生、学生、机器的工作小时数，每小时的代价是 5、3、8 美元。因为工作时间不能为负数，所以 $x_1,x_2,x_3 \ge 0$。研究生和学生每小时解决 1 个问题，机器每小时解决 2 个问题，总共有 4 个问题，所以 $x_1+x_2+2x_3=4$。目标：最小化 $c^T x$。

如果 3 个对象都在工作，那么 4 个问题需要 1 小时：$x_1=x_2=x_3=1$，代价是 $5+3+8=16$。但根本不需要研究生——学生性价比更高。让学生工作 2 小时、机器 1 小时，代价是 $6+8=14$。参见 Fig8.6，此时我们在边 QR 上（$x_1=0$）。但最佳点是 Q 点——全部由学生完成，代价 12。

**当 $Ax=b$ 只有 1 个方程，顶点 $(0,4,0)$ 只有 1 个非 0 分量。当 $Ax=b$ 有 m 个方程，顶点有 m 个非 0 分量。** 我们是对这 m 个变量求解 $Ax=b$ 的，**有 $n-m$ 个自由变量被设为 0。** 注意和之前的解方程不同：**我们不知道怎么选出这 m 个变量。**

**可能出现的顶点的个数，就是我们可以从 n 选出 m 的方法数：** $\binom{n}{m}$。如 $n=20, m=9$，那么可行集合会有约 $5\times10^9$ 种选择方法！对于 3 个顶点我们可以逐一验证，但对于 5 亿个顶点肯定行不通。

### 对偶问题

在 LP，问题总是成对出现的——会同时有一个最小化问题和一个最大化问题，也就是**原始问题**和它的**对偶问题**。
- 原始问题由矩阵 A 和两个向量 b、c 描述
- 对偶问题转置 A 并切换 b、c 向量：最大化 $b \cdot y$

对于上面的问题，一个作弊者提供收费给答案的服务。每个答案 y 美元，那么 4 个问题的答案就是 $4y$（注意，此时 $b=4$ 变成了耗费）。费用 y 必须小于等于研究生、学生、机器的话费：$y \le 5$ **和** $y \le 3$ **和** $2y \le 8$（注意，现在 $c=(5,3,8)$ 变成了不等式约束）。作弊者想要最大化收费 $4y$：

$$\text{Dual Problem:}\qquad \text{限制在 } A^T y \le c \text{ 的条件下，最大化 } b \cdot y$$

当 $y=3$，可以取得最大的收入 $4y=12$。对偶问题当中的最大值 12 和原始问题当中的最小值 12 是相等的。**最大=最小就是所谓的对偶性：只要原始或对偶问题其中之一有最佳解 $x^*$ 或 $y^*$，那么另外一个必定也有最佳解，而且原始问题当中的最小代价 $c \cdot x^*$ 必定等于对偶问题当中的最大收入 $b \cdot y^*$。**

证明较容易的前半部分——作弊者的收入 $b^T y$ 不能超过代价：

$$b^T y = (A x)^T y = x^T (A^T y) \le x^T c \tag{1}$$

最后一步 $\le$ 是因为 $x \ge 0$ 且 $s = c - A^T y \ge 0$，所以 $x^T s \ge 0$。完整的对偶定理：当 $b^T y$ 达到最大值，$x^T c$ 也达到最小值，且它们相等：$b \cdot y^* = c \cdot x^*$。当想要取得等号需要 $x^T s = 0$，所以最优解是：每一个 j，都有 $x_j^*=0$ 或 $s_j^*=0$。

## 4. 单纯形法（Simplex Method）

消去是求解线性方程的主力，TSM 是求解线性不等式的主力。TSM 从一个顶点移动到代价更小的另一个**相邻**顶点，最终（实践当中非常快）到达最小耗费的顶点。

**顶点：** 一个向量 $x \ge 0$，满足 m 个方程 $Ax=b$，并且**至多 m 个分量是正值，其他 $n-m$ 个分量都是 0**（0 分量就是自由变量，回代可得到主变量，而且 x 的所有分量必须非负）。

**相邻顶点：** 当前顶点 x 的一个分量从 0 变为正值——**进入变量**（entering variable），一个分量从正值变为 0——**离开变量**（leaving variable）。

**TSM 必须确定，哪个分量从 0 变为正值（进入变量）、哪个分量从正值变为 0（离开变量）。经过这样的分量变换之后，总代价必须降低，这就是 TSM 向 $x^*$ 逼近的一个步骤。**

整个思想：观察当前顶点存在的所有 0 分量，选择**其中一个** 0 分量从 0 变为 1，**其他非 0 分量必须做出调整来保持 $Ax=b$**。分量变换之后，回代方程得到新的顶点 x，然后用这个顶点 x 计算总代价 $c \cdot x$，和上一次的代价做减法得到**削减代价 r**。而给出了最大负值 r 的变量交换，即表示此次交换成功。

**例 1：** 以 Fig8.6 为例，假设当前顶点是 $P=(4,0,0)$——研究生完成所有问题，代价 20。如果学生工作 1 小时，现在 $x=(3,1,0)$，代价 18 → 削减代价 $r=-2$。如果机器工作 1 小时，$x=(2,0,1)$ 的代价也是 18 → 削减代价 $r=-2$。这种情况下 TSM 选择学生或机器作为进入变量都可以。

就算在这个简单的例子，第一步也很难得到最佳的 $x^*$。TSM 确实可以选择进入变量，但进入变量应该设为多少呢？例子当中，我们是通过将进入变量从 0 变为 1 来计算 r 的，但递增 1 个单位可能太少，也可能太多！现在假设 TSM 选择将研究生作为离开变量，所以会移动到 Q 或者 R 顶点。
