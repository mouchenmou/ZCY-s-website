

在 MkDocs Material 中渲染 Markdown 笔记时必须遵守的格式规则。

---

1. **在 callout 块（如 `!!! explanation`, `!!! example`, `!!! proof` 等）中写列表时**：列表前必须有一个空行。
   ```
   !!! explanation "标题"
       正文内容。

       - 列表项1
       - 列表项2
   ```
   正文文字和列表之间必须有空行，否则 Admonition 中列表渲染会出错。

2. **在 callout 块中写块级公式（`$$...$$`）时**：公式前后都必须各有一个空行。
   ```
   !!! example "标题"
       正文。

       $$
       w_k = \sqrt[n]{|z|} \cdot e^{i\frac{\arg z + 2k\pi}{n}}
       $$

       其中 $k = 0, 1, 2, \ldots, n-1$。
   ```
   如果 `$$` 前面没有空行，MkDocs 无法正确渲染公式块。

3. **不要用表格来罗列性质列表**：用编号列表（1. 2. 3.）而不是表格。表格只适合真正的二维对比数据。

4. **粗体/冒号行后面跟列表（`-` 或 `1.`），中间必须空一行**：这是 MkDocs 渲染的要求，不加空行列表无法渲染。编号列表 `1.` 和项目符号 `-` 都需要这条规则。
   - ❌
     ```
     **方法 1（线性扫描）：** 共 $V$ 轮：
     - 扫描所有未知顶点
     - 更新邻居
     ```
   - ❌
     ```
     **例 1：** S 是 9 维空间中 6 维子空间。
     1. 和 S 正交的子空间维数可能？→ 0, 1, 2, 3
     2. $S^\perp$ 维数？→ 一定是 3
     ```
   - ✅
     ```
     **方法 1（线性扫描）：** 共 $V$ 轮：

     - 扫描所有未知顶点
     - 更新邻居
     ```
   - ✅
     ```
     **例 1：** S 是 9 维空间中 6 维子空间。

     1. 和 S 正交的子空间维数可能？→ 0, 1, 2, 3
     2. $S^\perp$ 维数？→ 一定是 3
     ```
   - 这条规则适用于全文件所有位置（callout 内外都适用）。

5. **块级公式 `$$...$$` 上下都必须空一行**：不在 callout 里也一样。MkDocs 要求 `$$` 所在的上下行必须是空行，否则公式块渲染会出问题。
   - ✅
     ```
     正文文字。

     $$
     A^{-1}Ax = A^{-1}b \;\Longrightarrow\; x = A^{-1}b
     $$

     继续正文。
     ```
   - ❌
     ```
     正文文字：
     $$
     A^{-1}Ax = A^{-1}b \;\Longrightarrow\; x = A^{-1}b
     $$
     继续正文。
     ```
   简单记：`$$` 前一行的末尾和 `$$` 后一行的开头都必须空一行。
