# 🎓 雕虫小技

## 📚 板块介绍

作为大学牲还是要会一点雕虫小技的，比如安装Claude code，安装codex，安装Openclaw等等，现在没有AI大家都活不下去。Openclaw比较危险，不建议装在主机，我的Openclaw就没有装在Windows中，我怕它在Windows里面搞破坏。

### 🏫 技能列表

- Claude code安装
- 将AI接入Obsidian
- Codex安装

### 💪踩过的坑

在Claude Code中踩过的坑我真的算是非常非常有发言权的，好几天因为它奋战到深夜，希望你们不要重蹈我的覆辙啊。

1. 我原本是把Claude Code安装在Windows的主系统中，但它在Windows下真的很笨，本身就更适配Linux。而且把AI工具装在主系统中会有安全隐患，所以我选择把它迁到WSL中，顺便把WSL从C盘迁移到了D盘。过程中也踩了不少的坑，具体的步骤可以参考[Windows Claude Code 国内安装教程以及Ubuntu的重新配置](../笔记本/Claude%20code使用/Windows%20Claude%20Code%20国内安装教程以及Ubuntu的重新配置.md)。
2. 我原本以为Claudian插件不走本地的claude可执行文件，而是通过环境变量直接把API请求发送到云端。所以我直接把Windows主系统上的Claude Code删掉了，结果Claudian死活用不了，调教了很久依然失败。98上的朋友上说跨系统操作文件会有很多额外的耗时，所以最终无奈在主系统中又装回了Claude Code，但是除了Claudian需要用到主系统中的Claude Code，其余时间我用的都是WSL中的Claude Code。
3. 如果想在Terminal插件中跟claude code对话的话需要把默认的Shell改成WSL的bash，这样太麻烦了，因为Gemini Cli还得在Powershell中对话，所以不如Terminal插件用来跟Gemini Cli对话，Claudian用来跟Claude对话。
